# Coordination Rules

## Agent Communication
- 每个代理负责特定领域（脚本、UI、音频等）
- 跨领域修改需要明确说明影响范围
- 使用清晰的接口定义代理间依赖

## Task Flow
1. **分析** - 理解需求和影响范围
2. **规划** - 列出需要修改的文件
3. **征询** - 向用户展示计划并获得批准
4. **执行** - 按批准的计划进行修改
5. **验证** - 确认修改符合预期

## Quality Gates
- 代码必须符合 C# 规范
- 公共 API 需要 XML 文档注释
- 避免循环依赖
- 性能敏感代码需要说明理由
