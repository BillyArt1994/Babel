# 敌人生成系统

> **状态**: 已完成
> **作者**: 用户 + Claude
> **最后更新**: 2026-03-28
> **实现支柱**: 支柱 2（压力递增的绝望感）、支柱 6（倒金字塔的绝望螺旋）

## 概述

敌人生成系统根据一张可配置的「生成事件表」，在 15 分钟游戏过程中按时间轴持续生成人类单位。每条事件记录定义触发时机、可选敌人池、数量范围、生成侧，并支持一次性触发或按间隔重复触发。系统每帧扫描未触发的事件，条件满足时从对象池借出实例并交给敌人 AI 系统驱动。

## 玩家幻想

玩家不会直接感受到这个系统的存在，但会感受到它带来的节奏变化——"前两分钟游刃有余，然后精英单位突然出现……再然后祭司和工程师混进来，我开始顾不过来了"。生成系统是 15 分钟压力曲线的导演，它决定什么时候给玩家喘息空间，什么时候拧紧螺丝。

## 详细设计

### 核心规则

**生成事件数据结构（`SpawnEvent`）：**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `startTime` | `float` | 首次触发时间（秒，相对游戏开始） |
| `enemyPool` | `EnemyData[]` | 可选敌人类型列表，每次触发从中随机选一种 |
| `countMin` | `int` | 每次触发最少生成数量 |
| `countMax` | `int` | 每次触发最多生成数量（含，均匀随机） |
| `spawnSide` | `SpawnSide` (enum) | `Left` / `Right` / `Both` / `Random`（每个单位独立随机） |
| `repeat` | `bool` | 是否重复触发 |
| `interval` | `float` | 重复间隔（秒），仅 `repeat=true` 有效 |
| `endTime` | `float` | 重复结束时间（秒），-1 表示持续到游戏结束，仅 `repeat=true` 有效 |

**事件表（`SpawnTimeline` ScriptableObject）：**
- 一个 `List<SpawnEvent>` 存储在 ScriptableObject 中，路径 `Assets/Data/SpawnTimelines/default.asset`
- 系统启动时加载配置，事件按 `startTime` 排序缓存
- 支持替换整张配置表以切换难度模式（如 Easy / Normal / Hard）

**每帧执行逻辑：**
1. 查询 `GameLoopManager.IsPlaying()`，非进行中则跳过
2. 获取 `elapsedTime = GameLoopManager.GetElapsedTime()`
3. 遍历所有事件，检查是否满足触发条件（见公式节）
4. 满足触发条件的事件：随机决定本次生成的敌人类型和数量，调用对象池生成实例
5. 将生成的 `GameObject` 引用通知敌人 AI 系统，启动单位行为

**生成位置规则：**
- 左侧生成点：屏幕左边缘外 `SPAWN_OFFSET_X` 距离，Y 轴 = `LAYER_BASE_HEIGHT`（地面）
- 右侧生成点：屏幕右边缘外 `SPAWN_OFFSET_X` 距离，Y 轴 = `LAYER_BASE_HEIGHT`
- `Both`：本次生成的单位均分左右（奇数时多余的一个随机分配）
- `Random`：每个单位独立随机选左或右
- 所有单位在地面高度生成，由敌人 AI 驱动寻路至当前活跃层高度

**参考事件配置（默认 `default.asset`，需原型测试后调整）：**

|---|-----------|-----------|-------|-----------|--------|----------|---------|

| 2 | 120s | [Worker] | 2~3 | Both | ✅ | 3.5s | 300s |

### 状态与转换

| 事件状态 | 描述 | 进入条件 | 退出条件 |
|---------|------|---------|---------|
| 待触发 | 尚未到达 `startTime` | 初始 | `elapsedTime >= startTime` |
| 已触发（一次性） | 已执行，不再检查 | `repeat=false` 且条件满足 | 永久（终态） |
| 活跃中（重复） | 按 `interval` 周期触发 | `repeat=true` 且 `elapsedTime >= startTime` | `elapsedTime >= endTime` 或游戏结束 |
| 已结束（重复） | 超过 `endTime`，停止触发 | `elapsedTime >= endTime` | 永久（终态） |

### 与其他系统的交互

| 系统 | 方向 | 接口 |
|-----|------|------|
| 游戏循环管理器 | 输入 ← | `IsPlaying()` — 过滤非游戏中状态；`GetElapsedTime()` — 事件触发时机判断 |
| 对象池系统 | 输出 → | `Get(EnemyData, Vector3 spawnPos)` — 借出单位实例 |
| 敌人 AI 系统 | 输出 → | 生成完成后将 `GameObject` 引用传递，通知开始寻路行为 |
| 敌人数据库 | 输入 ← | `SpawnEvent.enemyPool` 直接引用 `EnemyData` 资产（Inspector 配置） |

## 公式

1. **一次性事件触发条件**
   ```
   shouldTrigger = (elapsedTime >= event.startTime) AND (event.triggered == false)
   触发后：event.triggered = true
   ```

2. **重复事件触发条件**
   ```
   isActive      = (elapsedTime >= event.startTime)
                   AND (event.endTime == -1 OR elapsedTime < event.endTime)
   shouldTrigger = isActive AND (elapsedTime >= event.nextTriggerTime)
   触发后：event.nextTriggerTime += event.interval
   ```
   - `nextTriggerTime` 初始值 = `startTime`

3. **本次生成数量（均匀随机整数）**
   ```
   count = Random.Range(event.countMin, event.countMax + 1)  // 含两端
   ```

4. **生成位置**
   ```
   spawnX_Left  = Camera.ScreenToWorldPoint(Vector3(0, 0, 0)).x  - SPAWN_OFFSET_X
   spawnX_Right = Camera.ScreenToWorldPoint(Vector3(Screen.width, 0, 0)).x + SPAWN_OFFSET_X
   spawnY       = LAYER_BASE_HEIGHT  // 所有单位在地面生成，由 AI 寻路到目标层
   ```

## 边缘情况

1. **对象池返回 null（展层增长期间的极短延迟）**
   - 处理：理论上对象池展层增长是同帧完成的，不会返回 null；若返回 null 则跳过本次生成并打印警告

2. **多个事件同帧触发（Boss 波涌现）**
   - 处理：允许，逐条处理，多条事件的总生成量可在同帧完成；这是合法的设计意图

3. **游戏暂停时计时行为**
   - 处理：`nextTriggerTime` 基于 `elapsedTime`（游戏时间），暂停时 `elapsedTime` 不增加，计时自动暂停，无需额外处理

4. **`endTime` 到达时有单位仍在途**
   - 处理：已在途单位不受影响，继续前进；`endTime` 仅控制是否继续触发新生成

5. **`enemyPool` 中类型的 `spawnStartTime` 与事件 `startTime` 不一致**
   - 情况：配置表事件在 `t=120s` 触发，但 `enemyPool` 中包含 `spawnStartTime=300s` 的 Priest
   - 处理：生成系统不二次检查 `EnemyData.spawnStartTime`；`OnValidate()` 在保存配置时对此发出警告
   - 原因：配置表是唯一控制面板，双重过滤增加复杂性

6. **`SpawnTimeline` 为空**
   - 处理：系统正常运行，不生成任何单位；打印警告日志（可用于测试其他系统时禁用敌人）

7. **`countMin > countMax`（配置错误）**
   - 处理：`OnValidate()` 检测并报错；运行时使用 `countMin` 作为固定值

## 依赖关系

**上游依赖（此系统依赖的系统）：**

| 系统 | 依赖类型 | 接口 |
|-----|---------|------|
| 游戏循环管理器 | 硬依赖 | `IsPlaying()`、`GetElapsedTime()` |
| 敌人数据库 | 硬依赖 | `EnemyData` 资产（通过 `SpawnEvent.enemyPool` 引用） |
| 对象池系统 | 硬依赖 | `Get(EnemyData, Vector3)` |

**下游依赖（依赖此系统的系统）：**

| 系统 | 依赖类型 | 接口 |
|-----|---------|------|
| 敌人 AI 系统 | 硬依赖 | 接收生成完成的 `GameObject` 引用，启动单位行为 |

## 调优参数

**所有事件参数均在 `SpawnTimeline` ScriptableObject 中配置，无需改代码。**

| 参数 | 位置 | 调整效果 |
|-----|------|---------|
| 各事件 `interval` | SpawnTimeline | 减小 → 密度↑，压力↑ |
| 各事件 `countMin/Max` | SpawnTimeline | 增大 → 每次涌来更多，突发压力↑ |
| 阶段 `startTime` | SpawnTimeline | 提前 → 难度前移，曲线变陡 |
| `SPAWN_OFFSET_X` | Inspector 常量 | 增大 → 生成点更远，玩家反应时间↑（难度↓） |

**多套难度配置：** 创建 `easy.asset` / `normal.asset` / `hard.asset` 三张不同 `SpawnTimeline`，在主菜单选择难度时切换即可。

## 验收标准

1. `t=0s` 时 Worker 开始从左右两侧按配置间隔生成
2. `t=120s` 时 Elite 按 `interval=15s` 开始周期出现
3. `t=300s` 时 Priest 和 Engineer 开始出现
4. `t=870s` 时生成密度明显提升（Boss 波），敌人数量接近对象池上限
5. 游戏暂停时生成停止，恢复后从暂停点继续，无跳帧生成
6. `countMin/countMax` 范围内的随机分布在多次测试中均匀
7. `SpawnTimeline` 为空时，游戏正常运行，控制台有警告
8. 替换为 `easy.asset` 或 `hard.asset` 后，生成行为完全切换
