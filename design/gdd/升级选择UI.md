# 升级选择 UI

> **状态**: 设计中
> **作者**: 用户 + Claude
> **最后更新**: 2026-03-29
> **实现支柱**: 支柱 3（Build 策略深度）、支柱 5（简洁的复杂性）

## 概述

升级选择 UI 是每次升级触发时弹出的全屏模态界面。场景背景被半透明暗色遮罩覆盖，屏幕中央水平排列 3 张技能卡片。每张卡片显示技能图标、名称、类型标签（点击形态/被动增强）、以及下一级的效果描述（若玩家已有该技能则显示"Level X → X+1"及效果变化）。玩家点击任意卡片即完成选择，界面消失，游戏继续。设计极简，决策快速。

## 玩家幻想

每次升级弹出是 15 分钟局内最重要的"策略时刻"——游戏暂停，世界安静下来，玩家看着三张卡片做决定。好的升级界面应让玩家在 2-3 秒内理解每张卡片的价值，并在 5-10 秒内做出决策，然后毫不犹豫地点击。信息量刚好：不过载、不模糊。服务支柱 3（Build 策略深度）。

## 详细设计

### 核心规则

1. 监听 `UpgradeEvents.OnOptionsGenerated(SkillData[] options)` 事件
2. 事件触发时，显示遮罩 + 从中央淡入 3 张卡片（若选项不足 3 个，则显示实际数量，最少 1 张）
3. 根据 SkillData 及当前技能状态填充每张卡片内容：
   - 玩家**未有**该技能：显示 "Level 1" 效果描述
   - 玩家**已有**该技能：显示 "Level X → X+1" 及下一级效果描述与数值变化
4. 玩家点击任意卡片 → 调用 `UpgradeSystem.SelectOption(cardIndex)` → 关闭界面
5. 选择后立即隐藏卡片，遮罩淡出，游戏自动恢复（由 UpgradeSystem → GameLoopManager 处理）

### 状态与转换

| 状态 | 显示 | 进入条件 | 退出条件 |
|------|------|---------|---------|
| 隐藏 | 不可见 | 初始 / 选择完成 | `OnOptionsGenerated` 事件 |
| 显示中 | 遮罩 + 卡片 | `OnOptionsGenerated` 事件 | 玩家点击任意卡片 |

### 与其他系统的交互

| 系统 | 方向 | 接口 |
|------|------|------|
| 升级系统 | 输入 ← | 监听 `UpgradeEvents.OnOptionsGenerated(SkillData[])` |
| 升级系统 | 输出 → | 调用 `UpgradeSystem.SelectOption(int index)` |
| 技能系统 | 输入 ← | `GetSkillLevel(SkillData)` — 查询玩家当前该技能的级数（用于显示 "Level X → X+1"） |

## 公式

无数值计算。卡片内容来源：
- 效果描述：`SkillData.description`（每级维护独立描述，或由级数 index 查询描述数组）
- 级数显示：`string.Format("Level {0} → {1}", currentLevel, currentLevel + 1)`
- 首次获得时：直接显示 `SkillData.description[0]`（Level 1 描述）

## 边缘情况

1. **选项数量 < 3**：升级系统保证最少 1 张；UI 水平居中显示实际数量，无需填充空白卡片
2. **选项数量 = 0**：升级系统此时不触发事件，UI 不受影响
3. **淡入动画期间点击**：淡入开始后立即启用点击响应，不等动画完成，保持快速响应
4. **同一张卡片快速双击**：第一次点击后立即隐藏 UI，后续点击因 UI 不可见而无效

## 依赖关系

**上游依赖：**

| 系统 | 依赖类型 | 接口 |
|-----|---------|------|
| 升级系统 | 硬依赖 | 监听 `UpgradeEvents.OnOptionsGenerated(SkillData[])`；调用 `SelectOption(int)` |
| 技能系统 | 软依赖 | `GetSkillLevel(SkillData)` — 查询已有技能的当前级数 |

**下游依赖：** 无（终端系统）

## 调优参数

| 参数名 | 默认值 | 安全范围 | 说明 |
|--------|--------|---------|------|
| `OVERLAY_ALPHA` | 0.7 | [0.4, 0.9] | 背景遮罩透明度，越高场景越暗 |
| `CARD_FADE_DURATION` | 0.2s | [0.1, 0.5] | 卡片淡入时长，过长会让玩家等待感强烈 |

## 视觉/音频需求

- 背景遮罩：半透明黑色，`OVERLAY_ALPHA = 0.7`
- 卡片淡入动画（0.2s）
- 选中卡片时：短暂高亮/缩放反馈（0.1s），然后消失
- 可选：升级弹出时播放一个短促的「升级」音效（由音效系统#20 实现，MVP 可跳过）

## UI 需求

**卡片结构（每张）：**
```
┌─────────────────────┐
│    [图标 64×64]      │
│    技能名称（大字）   │
│    [类型标签]        │   "点击形态" / "被动增强"（小标签）
│    ─────────────    │
│    Level 1 → 2      │   （已有时显示；首次则隐藏此行）
│    效果描述（小字）   │
└─────────────────────┘
```

**整体布局：**
- Canvas：Screen Space - Overlay，层级高于 HUD
- 遮罩层：全屏半透明黑色 Image
- 卡片容器：HorizontalLayoutGroup，居中对齐，间距 40px
- 卡片尺寸：约 160×220px（具体由美术调整）

## 验收标准

1. 升级触发时，遮罩和 3 张卡片正确出现，背景场景暗化
2. 每张卡片正确显示对应 SkillData 的图标、名称、类型标签
3. 已有技能的卡片正确显示"Level X → X+1"和下一级描述
4. 首次获得的技能卡片不显示"Level X"行
5. 点击任意卡片后，界面消失，游戏恢复
6. 快速双击卡片不导致 `SelectOption` 被调用两次
7. 选项少于 3 张时，卡片居中显示，布局不破坏

## 待解决问题

1. **技能多级描述的数据结构**：SkillData 当前设计中 `description` 是单字段，需确认是否改为数组（每级一条）或以其他方式支持多级描述。影响升级系统和技能数据库 GDD，需在实现前统一。
2. **`GetSkillLevel(SkillData)` 接口**：技能系统 GDD 中目前没有此方法（只有 `GetPassiveStacks()`），实现时需在技能系统中新增或改名，使其返回语义上的"级数"而非"叠加层数"。
3. **卡片进化链显示**：若技能有进化来源（`upgradesFrom != null`），是否在卡片上标注"进化自：XXX"？当前设计未包含，可在 UI 细化阶段讨论。
