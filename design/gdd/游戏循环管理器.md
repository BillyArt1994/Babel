# 游戏循环管理器

> **状态**: 已完成
> **作者**: 用户 + Claude
> **最后更新**: 2026-03-28
> **实现支柱**: 支柱 2（压力递增的绝望感）、支柱 6（倒金字塔的绝望螺旋）

## 概述

游戏循环管理器是游戏的核心框架系统，负责管理整个游戏会话的生命周期。它从游戏开始时启动 15 分钟倒计时，追踪游戏状态（进行中/暂停/胜利/失败），并在满足胜利或失败条件时结束游戏。

玩家通过暂停键可以随时暂停游戏，暂停时游戏时间停止，屏幕显示暂停弹窗。游戏的胜利条件是成功守住 15 分钟（倒计时归零），失败条件是塔完全建成（塔建造系统报告 100% 完成）。

这个系统不直接与玩家交互，而是作为基础设施为其他 9 个系统提供时间、状态和生命周期管理服务。

## 玩家幻想

这是一个基础设施系统，玩家不会直接"感受"到它的存在，但它支撑着两个核心情感体验：

1. **压力递增的紧张感** - 倒计时不断减少，配合塔的不断升高，创造"时间不多了"的紧迫感。玩家看着剩余时间从 15:00 倒数到 0:00，每一秒都在与人类的建造速度赛跑。

2. **守护尊严的成就感** - 当倒计时归零、塔未完成时，玩家获得"成功阻止了人类僭越"的胜利感。这个系统通过明确的胜利/失败判定，将抽象的"守护天神尊严"转化为具体的成就时刻。

参考游戏：Vampire Survivors 的倒计时系统 - 玩家清楚知道"再撑 X 分钟就赢了"，这种明确的目标创造了强烈的动机。

## 详细设计

### 核心规则

**游戏状态机：**
游戏循环管理器维护以下状态：
1. **未开始 (NotStarted)** - 初始状态
2. **进行中 (Playing)** - 游戏正在运行，时间流逝
3. **暂停 (Paused)** - 玩家手动暂停，时间停止
4. **升级选择中 (LevelingUp)** - 升级系统触发技能选择，时间停止
5. **胜利 (Victory)** - 倒计时归零，塔未完成
6. **失败 (Defeat)** - 塔完全建成

**计时规则：**
1. 游戏开始时，倒计时设为 900 秒（15 分钟）
2. 在"进行中"状态下，每帧减少 `Time.deltaTime`
3. 在"暂停"状态下，计时器不变
4. 倒计时归零时，触发胜利条件检查

**状态转换规则：**
1. 从"未开始"到"进行中"：用户点击开始游戏
2. 从"进行中"到"暂停"：用户按下暂停键（ESC）
3. 从"暂停"到"进行中"：用户再次按下暂停键或点击继续按钮
4. 从"进行中"到"胜利"：倒计时 <= 0 且塔未完成
5. 从"进行中"到"失败"：塔建造系统报告完成度 >= 100%

**胜利/失败判定：**
- 每帧检查：倒计时 <= 0 → 触发胜利
- 监听事件：塔建造系统发送"塔完成"事件 → 触发失败

### 状态与转换

| 当前状态 | 触发条件 | 目标状态 | 副作用 |
|---------|---------|---------|--------|
| 未开始 | 用户点击"开始游戏" | 进行中 | 初始化倒计时为 900 秒，发送 `OnGameStart` 事件 |
| 进行中 | 用户按下 ESC 键 | 暂停 | 停止时间流逝，显示暂停弹窗，发送 `OnGamePaused` 事件 |
| 进行中 | 升级系统触发升级 | 升级选择中 | 停止时间流逝，显示升级选择 UI，发送 `OnLevelUpStart` 事件 |
| 暂停 | 用户按下 ESC 键或点击"继续" | 进行中 | 恢复时间流逝，隐藏暂停弹窗，发送 `OnGameResumed` 事件 |
| 升级选择中 | 用户选择技能 | 进行中 | 恢复时间流逝，隐藏升级选择 UI，发送 `OnLevelUpComplete` 事件 |
| 进行中 | 倒计时 <= 0 | 胜利 | 停止时间流逝，发送 `OnVictory` 事件，触发结算 UI |
| 进行中 | 塔完成度 >= 100% | 失败 | 停止时间流逝，发送 `OnDefeat` 事件，触发结算 UI |
| 胜利 | 用户点击"返回主菜单" | 未开始 | 清理游戏数据，加载主菜单场景 |
| 失败 | 用户点击"返回主菜单" | 未开始 | 清理游戏数据，加载主菜单场景 |
| 胜利 | 用户点击"重新开始" | 进行中 | 重置所有系统，重新初始化倒计时 |
| 失败 | 用户点击"重新开始" | 进行中 | 重置所有系统，重新初始化倒计时 |

**不允许的转换：**
- 从"升级选择中"到"暂停"（升级选择优先级更高）
- 从"胜利"或"失败"直接回到"暂停"或"升级选择中"
- 从"未开始"直接到"暂停"、"升级选择中"、"胜利"或"失败"

### 与其他系统的交互

**提供给其他系统的接口：**

1. **时间查询接口**
   - `float GetRemainingTime()` - 返回剩余秒数
   - `float GetElapsedTime()` - 返回已过秒数
   - `float GetGameProgress()` - 返回进度百分比 (0.0 - 1.0)

2. **状态查询接口**
   - `GameState GetCurrentState()` - 返回当前游戏状态
   - `bool IsPlaying()` - 游戏是否在进行中
   - `bool IsPaused()` - 游戏是否暂停
   - `bool IsGameOver()` - 游戏是否结束（胜利或失败）

3. **事件系统**
   - `OnGameStart` - 游戏开始时触发（敌人生成系统、技能系统监听）
   - `OnGamePaused` - 游戏暂停时触发（所有 UI 监听）
   - `OnGameResumed` - 游戏恢复时触发（所有 UI 监听）
   - `OnLevelUpStart` - 升级选择开始时触发（升级选择 UI 监听）
   - `OnLevelUpComplete` - 升级选择完成时触发（升级系统监听）
   - `OnVictory` - 胜利时触发（结算 UI、解锁系统、排行榜系统监听）
   - `OnDefeat` - 失败时触发（结算 UI、解锁系统、排行榜系统监听）

**从其他系统接收的接口：**

1. **塔建造系统**
   - 监听 `OnTowerCompleted` 事件 - 当塔完成度达到 100% 时触发失败

2. **升级系统**
   - 接收 `RequestLevelUp()` 调用 - 触发升级选择状态
   - 接收 `CompleteLevelUp()` 调用 - 从升级选择返回进行中状态

3. **主菜单 UI**
   - 接收"开始游戏"指令 - 从未开始转到进行中

4. **结算 UI**
   - 接收"重新开始"指令 - 重置游戏
   - 接收"返回主菜单"指令 - 返回主菜单

## 公式

**时间计算：**

1. **剩余时间更新**
   ```
   remainingTime = remainingTime - Time.deltaTime
   ```
   - `remainingTime`: 剩余秒数（初始值 900）
   - `Time.deltaTime`: Unity 帧间隔时间（秒）
   - 仅在状态为"进行中"时执行

2. **游戏进度百分比**
   ```
   gameProgress = (TOTAL_DURATION - remainingTime) / TOTAL_DURATION
   ```
   - `TOTAL_DURATION`: 常量 900 秒
   - `gameProgress`: 范围 [0.0, 1.0]
   - 用于 UI 显示和其他系统的难度曲线

3. **已过时间**
   ```
   elapsedTime = TOTAL_DURATION - remainingTime
   ```
   - `elapsedTime`: 已过秒数，范围 [0, 900]

**胜利/失败判定：**

4. **胜利条件**
   ```
   isVictory = (remainingTime <= 0) AND (towerCompletion < 100%)
   ```

5. **失败条件**
   ```
   isDefeat = (towerCompletion >= 100%)
   ```
   - `towerCompletion`: 从塔建造系统获取，范围 [0, 100]

## 边缘情况

1. **倒计时和塔完成同时触发**
   - 情况：倒计时归零的同一帧，塔完成度达到 100%
   - 处理：失败优先 - 如果塔完成，立即判定失败，不检查倒计时
   - 原因：符合游戏逻辑（人类成功建塔就是失败）

2. **暂停时塔完成**
   - 情况：游戏暂停时，塔建造系统发送完成事件（不应发生，但需防御）
   - 处理：忽略事件，仅在"进行中"状态监听塔完成事件
   - 原因：暂停时所有游戏逻辑应停止

3. **负数倒计时**
   - 情况：倒计时低于 0（如 -0.1 秒）
   - 处理：立即钳制为 0，触发胜利判定
   - 原因：避免 UI 显示负数时间

4. **快速重复暂停/恢复**
   - 情况：玩家快速按 ESC 键
   - 处理：添加 0.2 秒冷却时间，防止状态抖动
   - 原因：避免事件系统过载和 UI 闪烁

5. **游戏结束后的输入**
   - 情况：胜利/失败后，玩家仍按暂停键或其他操作
   - 处理：在"胜利"和"失败"状态下，忽略所有游戏输入（除了 UI 按钮）
   - 原因：防止意外状态转换

## 依赖关系

**上游依赖（此系统依赖的系统）：**
- 无 - 这是基础层系统，不依赖其他游戏系统

**下游依赖（依赖此系统的系统）：**

1. **塔建造系统**（硬依赖）
   - 接口：监听 `OnGameStart` 事件开始建造逻辑
   - 接口：查询 `IsPlaying()` 判断是否更新建造进度
   - 数据流：塔建造系统 → 游戏循环管理器（`OnTowerCompleted` 事件）

2. **敌人生成系统**（硬依赖）
   - 接口：监听 `OnGameStart` 事件开始生成敌人
   - 接口：查询 `GetGameProgress()` 调整生成难度
   - 接口：查询 `IsPlaying()` 判断是否生成新敌人

3. **技能系统**（硬依赖）
   - 接口：监听 `OnGameStart` 事件初始化技能
   - 接口：查询 `IsPlaying()` 判断技能是否生效

4. **升级系统**（硬依赖）
   - 接口：调用 `RequestLevelUp()` 触发升级选择状态
   - 接口：调用 `CompleteLevelUp()` 恢复游戏
   - 接口：查询 `IsPlaying()` 判断是否可以触发升级

5. **所有 UI 系统**（硬依赖）
   - 游戏 HUD：查询 `GetRemainingTime()` 显示倒计时
   - 升级选择 UI：监听 `OnLevelUpStart` 显示选择界面
   - 结算 UI：监听 `OnVictory` 和 `OnDefeat` 显示结算界面
   - 主菜单 UI：发送"开始游戏"指令

6. **解锁系统**（软依赖）
   - 接口：监听 `OnVictory` 和 `OnDefeat` 检查成就

7. **排行榜系统**（软依赖）
   - 接口：监听 `OnVictory` 记录分数

## 调优参数

| 参数名 | 默认值 | 安全范围 | 说明 |
|--------|--------|---------|------|
| `TOTAL_DURATION` | 900 秒 | [300, 1800] | 游戏总时长。低于 300 秒压力过大，高于 1800 秒过于冗长 |
| `PAUSE_COOLDOWN` | 0.2 秒 | [0.1, 0.5] | 暂停键冷却时间。过低会导致状态抖动 |

**极端行为：**
- `TOTAL_DURATION = 60` - 游戏过短，玩家无法体验完整循环
- `TOTAL_DURATION = 3600` - 游戏过长，违反"15 分钟一局"的设计支柱
- `PAUSE_COOLDOWN = 0` - 可能导致快速按键时状态抖动和事件风暴

**不可调参数（硬编码）：**
- 游戏状态枚举值
- 事件名称
- 胜利/失败判定逻辑

## 验收标准

**功能测试：**
1. 游戏开始时，倒计时从 15:00 开始递减
2. 按 ESC 键可以暂停游戏，时间停止
3. 再次按 ESC 或点击"继续"可以恢复游戏
4. 倒计时归零且塔未完成时，触发胜利
5. 塔完成度达到 100% 时，触发失败
6. 升级系统触发时，游戏自动暂停并显示选择界面
7. 选择技能后，游戏自动恢复

**事件测试：**
8. `OnGameStart` 事件在游戏开始时正确触发
9. `OnLevelUpStart` 和 `OnLevelUpComplete` 事件在升级流程中正确触发
10. `OnVictory` 和 `OnDefeat` 事件在游戏结束时正确触发

11. 快速按 ESC 键不会导致状态抖动
12. 倒计时不会显示负数

**性能标准：**
14. 状态检查和时间更新不超过 0.1ms/帧
