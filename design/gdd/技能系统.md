# 技能系统

> **状态：** 重构中
> **作者：** 独立开发者 + AI
> **最后更新：** 2026-04-17
> **实现支柱：** 支柱1（神罚节奏感）、支柱3（Build 策略深度）、支柱4（视觉冲击力）

## 概述

技能系统是游戏逻辑的核心枢纽，负责以下职责：

1. **管理玩家装备的 Skill 列表**，每个 Skill = 一个 ITrigger + 一个 IEffect
2. **通过 SkillFactory 组装 Skill 实例**，将 SkillConfig（静态配置）转化为可运行的战斗行为
3. **统一管理 Trigger 生命周期**（Enable / Disable / Update）
4. **委托 EffectManager 处理持续型效果**（DotEffect、BuffEffect）
5. **响应 GameLoopManager 状态变更**，集中控制所有 Trigger 的启停

本系统是将 SkillConfig（静态配置）转化为实际战斗行为的翻译层。唯一调用 `ClickAttackSystem.ExecuteAttack()` 的路径是各 IEffect 实现内部。SkillSystem 本身不直接操作攻击逻辑，而是通过 Trigger → Effect 回调链完成分发。

## 玩家幻想

服务支柱1（神罚节奏感）+ 支柱3（Build 策略深度）。

玩家感受到的是：每次点击的力量感随蓄力增强（蓄力满格时大伤害的爆发感），被动技能悄然叠加让同一次点击越来越"重"（数值增长的隐性成就感），连锁反应（余震爆炸、瘟疫扩散）带来出乎意料的蝴蝶效应快感。

技能系统不应让玩家意识到它的存在——它的目标是让"我选了这个 Build 所以打出了这个效果"的因果关系清晰可感。

## 详细设计

### 核心数据结构

#### ITrigger 接口

```csharp
public interface ITrigger {
    void Bind(Action<EffectContext> callback);  // 触发时调用
    void Enable();
    void Disable();
    void Update(float deltaTime);  // 供计时器类 Trigger 使用
}
```

#### IEffect 接口

```csharp
public interface IEffect {
    void Execute(EffectContext context);
}
```

#### EffectContext（参数包）

```csharp
public struct EffectContext {
    public Vector2 worldPos;       // 触发位置
    public float chargeRatio;      // 蓄力比（OnClickTrigger 填充）
    public IDamageable target;     // 命中目标（OnHitTrigger 填充）
    public bool isCrit;            // 是否暴击
    public float damageMultiplier; // 伤害倍率（由 BuffEffect 叠加）
}
```

#### Skill 类

```csharp
public class Skill {
    public SkillConfig Config { get; }
    public ITrigger Trigger { get; }
    public IEffect Effect { get; }

    // Trigger 回调时自动调用 Effect.Execute(context)
}
```

### 核心规则

#### A. Trigger 实现

**OnClickTrigger**

订阅 InputSystem 的 `OnMouseDown` / `OnMouseHeld` / `OnMouseUp`，处理蓄力计时、冷却管理、暴击判定，最终填充 EffectContext 并调用绑定的 callback。

参数：`cooldown`（冷却时间）、`chargeTime`（满蓄所需时长）。

```
OnMouseDown  → _isCharging = true；记录起始位置；广播 OnChargeStarted(worldPos)
OnMouseHeld  → 累加 holdDuration；广播 OnChargeUpdated(worldPos, holdDuration)
OnMouseUp    → 计算 chargeRatio；判定暴击；填充 EffectContext；调用 callback；重置冷却
```

冷却检查在 OnMouseUp 时执行：`_cooldownTimer > 0` 则静默丢弃，不排队。

蓄力比计算：
- `chargeTime > 0`：`chargeRatio = clamp(holdDuration / chargeTime, 0, 1)`
- `chargeTime == 0`：`chargeRatio = 1.0`（即时技能始终满伤）

暴击判定：`isCrit = Random.value < SkillSystem.GlobalCritChance`，结果写入 EffectContext。

**OnHitTrigger**

订阅 `ClickAttackSystem.OnAttackExecuted`，每次命中时按概率（`chance`）触发。EffectContext 中填充命中目标（`target`）和命中位置（`worldPos`）。用于余震（aftershock）。

参数：`chance`（触发概率 [0, 1]）。

**OnTimerTrigger**

在 `Update(deltaTime)` 中计时，按固定间隔（`interval`）触发。EffectContext 填充当前塔基位置。用于自动攻击被动（天雷、神火、追踪光球）。

参数：`interval`（触发间隔，秒）。

**OnKillTrigger**

订阅 `EnemyEvents.OnEnemyDied`，每次击杀时按概率（`chance`）触发。EffectContext 填充死亡位置（`worldPos`）。用于瘟疫（plague）、狂怒（rage）。

参数：`chance`（触发概率 [0, 1]，默认 1.0 表示每次击杀必触发）。

---

#### B. Effect 实现

**DamageEffect**

即时伤害。内部调用 `ClickAttackSystem.ExecuteAttack()`，组装 `AttackRequest`。

参数：`damage`（基础伤害）、`radius`（0 = 单体，>0 = AOE 半径）。

AttackType 推断规则（按优先级）：

| 条件 | AttackType |
|------|-----------|
| `chainCount > 0` | Chain |
| `piercing == true` | Pierce |
| `radius > 0` | AOE |
| 其他 | Single |

实际伤害计算（在 DamageEffect.Execute 中）：

```
finalDamage = config.damage
            × lerp(1.0, CHARGE_MULT, context.chargeRatio)
            × context.damageMultiplier
            × (context.isCrit ? CRIT_MULT : 1.0)
```

**DotEffect**

持续伤害。注册到 EffectManager，由 EffectManager 每帧 Tick，过期自动移除。

参数：`dps`（每秒伤害）、`duration`（持续时长，秒）。

**BuffEffect**

属性增益。修改玩家属性倍率（如 damageMult、cooldownReduction 等），持续 `duration` 秒后由 EffectManager 移除。叠加时刷新持续时间。

参数：`statName`（目标属性名）、`value`（修正量）、`duration`（持续时长，-1 = 永久）。

**SummonEffect**

召唤投射物或单位。

参数：`prefabPath`（预制体路径）、`count`（数量）。

---

#### C. SkillFactory 与反射组装

`SkillFactory` 接收 `SkillConfig`，通过 `TriggerFactory` 和 `EffectFactory` 组装出 `Skill` 实例：

- **TriggerFactory**：根据 `triggerType` 字符串反射创建 `ITrigger` 实例，从 `triggerParams` 字典填充参数（cooldown、chargeTime、interval、chance 等）。
- **EffectFactory**：根据 `effectType` 字符串反射创建 `IEffect` 实例，从 `effectParams` 字典填充参数（damage、radius、dps、duration 等）。

组装后调用 `trigger.Bind(effect.Execute)`，将两者绑定。

---

#### D. SkillSystem 生命周期管理

```csharp
// 持有当前装备的所有 Skill
private List<Skill> _skills = new();

void AddSkill(SkillConfig config) {
    var skill = SkillFactory.Create(config);
    skill.Trigger.Enable();
    _skills.Add(skill);
}

void RemoveSkill(SkillConfig config) {
    var skill = _skills.Find(s => s.Config == config);
    if (skill == null) return;
    skill.Trigger.Disable();
    _skills.Remove(skill);
}

void Update(float deltaTime) {
    foreach (var skill in _skills)
        skill.Trigger.Update(deltaTime);
}
```

`GlobalCritChance` 由 EffectManager 中所有活跃 `BuffEffect(statName="critChance")` 叠加后汇总，OnClickTrigger 每次触发时从 SkillSystem 取值。

---

#### E. EffectManager

管理所有持续型效果（DotEffect、BuffEffect）：

- `AddEffect(ITimedEffect effect)` — 注册活跃效果
- `Tick(float deltaTime)` — 每帧递减剩余时间，触发 DotEffect 伤害，过期自动移除
- `HasEffect(string effectId)` — 查询某效果是否存在
- `GetEffects<T>()` — 按类型获取活跃效果列表（供 SkillSystem 汇总 BuffEffect 属性）

### 状态与转换

**冷却维度（OnClickTrigger 内部）：**

```
Ready（_cooldownTimer <= 0）  ←──── 冷却到期
     ↓ OnMouseUp 触发
OnCooldown（_cooldownTimer > 0）──→ Update() 递减
```

**蓄力维度（OnClickTrigger 内部）：**

```
Idle ──OnMouseDown──→ Charging ──OnMouseUp──→ Idle
                           ↓
                     OnMouseHeld 广播蓄力进度
```

**游戏状态响应：**

SkillSystem 订阅 GameLoopManager 事件，统一控制所有 Trigger 的 Enable/Disable：

| GameLoopManager 状态 | 点击攻击 | 自动攻击 | 蓄力特效 |
|---------------------|---------|---------|---------|
| Playing | ✅ Trigger 已 Enable | ✅ 计时器运行 | ✅ 显示 |
| Paused | ❌ Trigger 已 Disable | ❌ 计时器暂停 | ❌ 停止 |
| LevelingUp | ❌ Trigger 已 Disable | ❌ 计时器暂停 | ❌ 停止 |
| Victory / Defeat | ❌ Trigger 已 Disable + 清空 | ❌ 停止 | ❌ 停止 |

状态切换通过订阅 `GameLoopManager.OnGamePaused` / `OnGameResumed` / `OnLevelUpStart` / `OnLevelUpComplete` / `OnVictory` / `OnDefeat` 实现，不在每帧轮询。

### 与其他系统的交互

| 方向 | 系统 | 接口 |
|------|------|------|
| 订阅 | 游戏循环管理器（#1） | `OnGamePaused`、`OnGameResumed`、`OnLevelUpStart`、`OnLevelUpComplete`、`OnVictory`、`OnDefeat` |
| 订阅（通过 OnClickTrigger） | 输入系统（#2） | `OnMouseDown`、`OnMouseHeld`、`OnMouseUp` |
| 读取 | 技能数据库（#3） | `SkillConfig` 所有字段（只读） |
| 调用（通过 DamageEffect） | 点击攻击系统（#8） | `ClickAttackSystem.ExecuteAttack(AttackRequest)` |
| 订阅（通过 OnHitTrigger） | 点击攻击系统（#8） | `OnAttackExecuted(AttackResult)` |
| 订阅（通过 OnKillTrigger） | 敌人（#10 定义） | `EnemyEvents.OnEnemyDied(EnemyData, Vector2)` |
| 被调用 | 升级系统（#11） | `AddSkill(SkillConfig)`、`RemoveSkill(SkillConfig)` |
| 广播（通过 OnClickTrigger） | 粒子特效系统（#12） | `OnChargeStarted(worldPos)`、`OnChargeUpdated(worldPos, ratio)` |
| 查询接口 | 游戏 HUD（#14） | `GetEquippedSkills()`、`GetCooldownProgress(skillId)` → [0, 1] |

**设计说明**：
`EnemyEvents.OnEnemyDied` 是全局静态事件，由各敌人在自身死亡时广播，统计系统也应订阅此事件用于结算计数。蓄力特效广播由 OnClickTrigger 内部完成，SkillSystem 不直接广播特效事件。

## 公式

**1. 蓄力伤害（OnClickTrigger + DamageEffect）**

```
chargeRatio  = clamp(holdDuration / chargeTime, 0, 1)  [chargeTime > 0 时]
             = 1.0                                      [chargeTime == 0 时]

finalDamage  = config.damage
             × lerp(1.0, CHARGE_MULT, chargeRatio)      // 蓄力加成：默认 1.5×
             × context.damageMultiplier                 // BuffEffect 叠加的伤害倍率
             × (context.isCrit ? CRIT_MULT : 1.0)       // 暴击：默认 2.0×
```

**2. 冷却时间（OnClickTrigger 内部）**

```
actualCooldown = config.cooldown × cooldownMult
               ≥ config.cooldown × MIN_COOLDOWN_MULT  (下限 0.2，防归零)

cooldownMult 由 EffectManager 中所有活跃 BuffEffect(statName="cooldownMult") 叠加汇总
```

**3. 伤害倍率（BuffEffect 叠加）**

```
context.damageMultiplier = 1.0
                         + Σ(BuffEffect(statName="damageMult").value × 活跃层数)
```

每次 DamageEffect.Execute 调用时，从 SkillSystem 查询 EffectManager 当前汇总的 damageMult 填入 EffectContext。

**4. 冷却缩减倍率（BuffEffect 叠加）**

```
cooldownMult = 1.0
             + Σ(BuffEffect(statName="cooldownMult").value)   // 负值表示缩减
cooldownMult = clamp(cooldownMult, 0.20, 1.0)
```

**5. 余震 / 瘟疫伤害（DamageEffect 参数）**

```
aftershockDamage = mainAttackConfig.damage × AFTERSHOCK_RATIO   // 默认 0.3
plagueDamage     = mainAttackConfig.damage × PLAGUE_RATIO        // 默认 0.2
```

余震和瘟疫均通过独立的 Skill（OnHitTrigger/OnKillTrigger + DamageEffect）实现，不共享主攻击的 EffectContext。

## 边界情况

| 情况 | 处理方式 |
|------|---------|
| 蓄力期间 GameState 变为 LevelingUp | SkillSystem 调用所有 Trigger.Disable()；OnClickTrigger.Disable() 内部广播 `OnChargeUpdated(pos, 0)` 取消特效，并重置 `_isCharging = false` |
| OnClickTrigger 蓄力期间被 Disable，之后 MouseUp 到达 | Disable 后 OnClickTrigger 不再响应输入事件（已取消订阅），MouseUp 静默丢弃 |
| `chargeTime == 0` 时计算 chargeRatio | 特判：直接赋值 1.0，不做除法，避免除以零 |
| OnHitTrigger 触发时又有 OnHitTrigger（余震触发余震） | DamageEffect 调用 ExecuteAttack 时传 `isPassiveAttack = true`；OnHitTrigger 检查此标记，被动攻击命中不再触发 OnHitTrigger，阻断递归 |
| 瘟疫链式传播杀死大量敌人（性能） | OnKillTrigger 天然受限于该帧死亡敌人数，无需额外限制 |
| OnTimerTrigger 在游戏暂停期间计时 | Disable 后 Update 不再被 SkillSystem 调用，计时自然暂停 |
| BuffEffect 狂怒减少冷却后 timer 为负 | OnClickTrigger 内部：`_cooldownTimer = max(0, _cooldownTimer - reduction)` |
| 首局游戏：AddSkill 调用前的初始状态 | `_skills` 列表为空，SkillSystem 在 GameLoopManager.OnGameStart 时通过 `AddSkill(starterConfig)` 初始化默认技能（神罚之指） |
| EffectManager 中同类 BuffEffect 叠加 | 叠加时刷新持续时间（取最长），value 相加；永久型（duration = -1）不过期 |

## 调节旋钮

| 旋钮 | 默认值 | 安全范围 | 极端行为 |
|------|--------|---------|---------|
| `CHARGE_MULT` | 1.5 | 1.0 – 3.0 | 1.0 = 蓄力无意义；>2.0 = 必须蓄力否则伤害太低 |
| `CRIT_MULT` | 2.0 | 1.5 – 4.0 | <1.5 = 暴击感觉不到；>3.0 = 暴击破坏平衡 |
| `MAX_CRIT_CHANCE` | 0.8 | 0.5 – 0.95 | 1.0 = 必暴击，失去随机感 |
| `MIN_COOLDOWN_MULT` | 0.2 | 0.1 – 0.5 | <0.1 = 几乎无冷却，画面刷爆 |
| `AFTERSHOCK_RATIO` | 0.3 | 0.1 – 0.6 | >0.6 = 余震比主攻伤害高，本末倒置 |
| `AFTERSHOCK_RADIUS` | 1.5 | 0.5 – 3.0 | >3.0 接近 AOE 主攻击范围，效果重叠 |
| `PLAGUE_RATIO` | 0.2 | 0.1 – 0.5 | >0.5 = 瘟疫链式传播秒杀所有人，破坏节奏 |
| `PLAGUE_RADIUS` | 2.0 | 1.0 – 4.0 | >4.0 = 一个死亡感染全屏 |
| `RAGE_COOLDOWN_REDUCTION` | 0.5 | 0.1 – 1.0 | >1.0 = 击杀后直接清空冷却 |

## 视觉/音效需求

本系统广播以下事件供特效系统消费（本系统不产生视觉，由 OnClickTrigger 内部广播）：

- `OnChargeStarted(Vector2 worldPos)` — 蓄力特效开始
- `OnChargeUpdated(Vector2 worldPos, float chargeRatio)` — 蓄力特效强度（0→1）
- 冷却进度通过 `GetCooldownProgress(skillId)` 接口由 HUD 轮询显示（而非事件推送）

## UI 需求

**游戏 HUD 需要的查询接口：**

```csharp
IReadOnlyList<Skill> GetEquippedSkills()             // 当前装备技能列表（图标 + 名称）
float GetCooldownProgress(string skillId)            // 0 = 可用，1 = 刚刚进入冷却
                                                     // 返回 _cooldownTimer / actualCooldown
                                                     // 由对应 OnClickTrigger 提供
```

## 验收标准

1. **Skill 组装**：给定 SkillConfig（triggerType = "OnClick"，effectType = "Damage"），SkillFactory 正确创建并绑定 Skill 实例，Trigger 和 Effect 均非 null
2. **点击攻击流程**：Playing 状态下，点击 → 蓄力 → 释放 → EffectContext 正确填充（chargeRatio、isCrit），DamageEffect 调用 ExecuteAttack，伤害符合蓄力公式
3. **冷却忽略**：冷却期间多次点击，仅在冷却结束后的下一次点击生效，中间点击静默丢弃（OnClickTrigger 内部验证）
4. **满蓄力伤害**：chargeRatio = 1 时，finalDamage = config.damage × CHARGE_MULT × damageMultiplier（暴击关闭）
5. **即时攻击伤害**：chargeTime = 0 时，chargeRatio 恒为 1.0，finalDamage = config.damage × CHARGE_MULT × damageMultiplier
6. **BuffEffect 叠加**：添加 2 个 BuffEffect(statName="damageMult", value=0.3)，汇总 damageMultiplier = 1.6，DamageEffect 伤害正确放大
7. **暴击**：GlobalCritChance = 1.0（测试配置）时，所有 OnClickTrigger 触发均产生 isCrit = true，finalDamage = ... × CRIT_MULT
8. **余震**：装备 OnHitTrigger + DamageEffect 余震 Skill → 每次命中触发二次 AOE；isPassiveAttack = true 的命中不再触发余震（防递归，最多一层）
9. **自动攻击**：OnTimerTrigger(interval=3s) → 每 3 秒触发一次 Effect.Execute（误差 < 1 帧），游戏暂停时计时停止
10. **OnKillTrigger**：击杀敌人 → 装备了 OnKillTrigger 的 Skill 触发 Execute；概率参数 chance = 0 时永不触发
11. **游戏状态响应**：Paused / LevelingUp 时 SkillSystem 调用所有 Trigger.Disable()，点击无效，计时暂停；Playing 时恢复 Enable
12. **Victory / Defeat 清空**：进入 Victory / Defeat 后，_skills 列表清空，所有 Trigger Disable，后续 Update 调用无副作用
13. **EffectManager 过期**：DotEffect(duration=2s) 注册后，2 秒后被 EffectManager 自动移除，HasEffect 返回 false
14. **性能**：每帧 SkillSystem.Update() 开销 < 0.1ms（含所有 OnTimerTrigger 递减）
