# 技能系统

> **状态：** 已完成
> **作者：** 独立开发者 + AI
> **最后更新：** 2026-03-28
> **实现支柱：** 支柱1（神罚节奏感）、支柱3（Build 策略深度）、支柱4（视觉冲击力）

## 概述

技能系统是游戏逻辑的核心枢纽，负责五件事：

1. **订阅输入事件**，捕获玩家点击意图
2. **管理点击形态**（互斥：同时只有一个），处理蓄力计算、冷却倒计时
3. **管理被动技能列表**，维护伤害/冷却综合倍率
4. **驱动自动攻击被动**（天雷、神火、追踪光球）的独立计时器
5. **响应击杀事件**，触发击杀类被动（瘟疫、狂怒）

本系统是唯一调用 `ClickAttackSystem.ExecuteAttack()` 的系统，也是将 SkillData（静态配置）转化为实际战斗行为的翻译层。

## 玩家幻想

服务支柱1（神罚节奏感）+ 支柱3（Build 策略深度）。

玩家感受到的是：每次点击的力量感随蓄力增强（蓄力满格时大伤害的爆发感），被动技能悄然叠加让同一次点击越来越"重"（数值增长的隐性成就感），连锁反应（余震爆炸、瘟疫扩散）带来出乎意料的蝴蝶效应快感。

技能系统不应让玩家意识到它的存在——它的目标是让"我选了这个 Build 所以打出了这个效果"的因果关系清晰可感。

## 详细设计

### 核心数据结构

```csharp
// 被动技能当前状态（含叠加层数）
private struct PassiveEntry {
    public SkillData data;
    public int       stacks;  // 当前叠加层数（1 起，上限 data.maxStackCount）
}

// 从被动列表计算出的综合修正值，每次 AddSkill 后刷新
private struct PassiveModifiers {
    public float damageMult;      // 伤害倍率，默认 1.0
    public float cooldownMult;    // 冷却时间倍率，默认 1.0（越小冷却越短）
    public float critChance;      // 暴击概率 [0, 0.8]
    public bool  hasExecute;      // 是否有处决被动
    public bool  hasAfterShock;   // 是否有余震被动
    public bool  hasPlague;       // 是否有瘟疫被动
    public bool  hasMark;         // 是否有神罚印记被动
    public bool  hasRage;         // 是否有狂怒被动
}
```

### 核心规则

#### A. 点击攻击流程

```
InputSystem.OnMouseDown(worldPos)
  → _isCharging = true
  → _chargeWorldPos = worldPos
  → 广播 OnChargeStarted(worldPos)（供粒子特效系统用）

InputSystem.OnMouseHeld(worldPos, holdDuration)
  → 广播 OnChargeUpdated(worldPos, holdDuration)（供蓄力特效用）

InputSystem.OnMouseUp(worldPos, holdDuration)
  → TryFireClickAttack(worldPos, holdDuration)
  → _isCharging = false
```

**`TryFireClickAttack(worldPos, holdDuration)` 详细步骤：**

1. 检查游戏状态：`GameLoopManager.GetCurrentState() != Playing` → 直接返回（忽略）
2. 检查冷却：`_cooldownTimer > 0` → 直接返回（忽略，冷却中不排队）
3. 计算蓄力比：  
   - 若 `_activeClickForm.chargeTime > 0`：`chargeRatio = clamp(holdDuration / chargeTime, 0, 1)`  
   - 若 `chargeTime == 0`：`chargeRatio = 1.0`（即时技能始终满伤）
4. 判定暴击：`isCrit = Random.value < _mods.critChance`
5. 计算实际伤害：
   ```
   baseDamage = skillData.damage
               × lerp(1.0, CHARGE_MULT, chargeRatio)
               × _mods.damageMult
               × (isCrit ? CRIT_MULT : 1.0)
   ```
6. 根据 SkillData 字段推断 AttackType（见下表），组装 `AttackRequest`
7. 调用 `ClickAttackSystem.ExecuteAttack(request)`
8. 重置冷却：`_cooldownTimer = skillData.cooldown × _mods.cooldownMult`

**AttackType 推断规则（按优先级）：**

| 条件 | AttackType |
|------|-----------|
| `chainCount > 0` | Chain |
| `piercing == true` | Pierce |
| `aoeRadius > 0` | AOE |
| 其他 | Single |

---

#### B. AttackRequest 组装（点击形态）

```csharp
var request = new AttackRequest {
    worldPos     = worldPos,
    attackType   = inferredType,
    damage       = baseDamage,
    radius       = skillData.aoeRadius,
    chainCount   = skillData.chainCount,
    chainRadius  = DEFAULT_CHAIN_RADIUS,    // ClickAttackSystem 的旋钮
    chainDecay   = DEFAULT_CHAIN_DECAY,     // ClickAttackSystem 的旋钮
    slowPercent  = skillData.slowPercent,
    slowDuration = skillData.slowDuration,
};
// 处决被动：标记到 request（ClickAttackSystem 不处理，由 IDamageable 实现）
// 处决逻辑转移到 TakeDamage：若目标血量 < 阈值，伤害×2
```

---

#### C. 被动技能管理

**`AddSkill(SkillData skill)` 规则：**

| 条件 | 行为 |
|------|------|
| `skill.skillType == ClickForm` | 替换 `_activeClickForm`，冷却重置为 0 |
| Passive，已在列表且未达叠加上限 | `stacks++` |
| Passive，已在列表且已达叠加上限 | 静默拒绝（升级系统不应再提供此选项） |
| Passive，首次添加 | 加入列表，`stacks = 1`；若为 AutoAttack，初始化计时器 |
| 任意情况 | 最后调用 `RecalculateModifiers()` |

**`RecalculateModifiers()` 规则：**

遍历所有 `PassiveEntry`，按类型累加：

| PassiveCategory | 字段用途 | 累加方式 |
|----------------|---------|---------|
| Frequency（神速、连击、狂怒） | `cooldown` 字段存储每层减少量（如 `−0.20`） | `cooldownMult += data.cooldown × stacks` |
| Power — 神力 | `damage` 字段存储每层加成（如 `0.30`） | `damageMult += data.damage × stacks` |
| Power — 暴击 | skillId = `"crit_strike"`，固定 +20% 每层 | `critChance += 0.20 × stacks` |
| Power — 处决 | skillId = `"execute"`，布尔标记 | `hasExecute = true` |
| Effect（余震/瘟疫/神罚印记） | 布尔存在性 | 设置对应 `has*` 标记 |
| AutoAttack | 无倍率贡献 | 仅由 Update() 定时器处理 |

后处理约束：
```
_mods.cooldownMult = clamp(_mods.cooldownMult, MIN_COOLDOWN_MULT, 1.0f)  // 最少 0.2
_mods.critChance   = clamp(_mods.critChance, 0, MAX_CRIT_CHANCE)          // 最多 0.8
```

---

#### D. 自动攻击被动（AutoAttack）

在 `Update()` 中驱动：

```csharp
foreach (var passive in _autoPassives) {
    _autoTimers[passive.data] -= Time.deltaTime;
    if (_autoTimers[passive.data] <= 0) {
        FireAutoAttack(passive.data);
        _autoTimers[passive.data] = passive.data.triggerInterval;
    }
}
```

**各自动攻击的目标选择：**

| 技能 | 目标选择逻辑 | AttackType |
|------|------------|-----------|
| 天雷 | 随机选取屏幕内一个活体敌人 | Single |
| 神火 | 以塔基位置为圆心，固定范围 AOE | AOE |
| 追踪光球 | 选取最近一个活体敌人（距离最小） | Single |

自动攻击调用 `ClickAttackSystem.ExecuteAttack()` 时，传入 `isPassiveAttack = true` 标记（在 AttackRequest 中添加此字段），供点击攻击系统和下游特效系统区分"主动点击"与"被动触发"。

---

#### E. 效果类被动（Effect Passives）

**余震（aftershock）— 订阅 `OnAttackExecuted`：**

```csharp
void OnAttackExecuted(AttackResult result) {
    if (_processingEffectPassives) return;  // 防递归
    if (!_mods.hasAfterShock) return;
    if (result.hits.Length == 0) return;

    _processingEffectPassives = true;
    foreach (var hit in result.hits) {
        ClickAttackSystem.ExecuteAttack(new AttackRequest {
            worldPos    = hit.hitPosition,
            attackType  = AOE,
            damage      = _aftershockDamage,   // 独立旋钮，默认 = 主攻击 × 0.3
            radius      = _aftershockRadius,   // 默认 1.5
            isPassiveAttack = true,
        });
    }
    _processingEffectPassives = false;
}
```

**瘟疫（plague）— 订阅 `EnemyEvents.OnEnemyDied`：**

```csharp
void OnEnemyDied(EnemyData data, Vector2 pos) {
    if (!_mods.hasPlague) return;
    // 瘟疫链式传播：死亡位置释放瘟疫 AOE（自然有限，无需限制层数）
    ClickAttackSystem.ExecuteAttack(new AttackRequest {
        worldPos    = pos,
        attackType  = AOE,
        damage      = _plagueDamage,    // 默认 = 主攻击 × 0.2（低伤高传播）
        radius      = _plagueRadius,    // 默认 2.0
        isPassiveAttack = true,
    });
}
```

**狂怒（rage）— 订阅 `EnemyEvents.OnEnemyDied`：**

```csharp
void OnEnemyDied(EnemyData data, Vector2 pos) {
    if (!_mods.hasRage) return;
    // 击杀后减少当前冷却
    _cooldownTimer = max(0, _cooldownTimer - RAGE_COOLDOWN_REDUCTION);
}
```

**神罚印记（mark）— 订阅 `OnAttackExecuted`：**  
在命中目标上添加印记（`IDamageable.AddMark()`），标记后目标受到额外伤害（由敌人在 TakeDamage 中检查自身是否被标记并乘以伤害系数）。MVP 阶段印记逻辑在敌人 AI 系统（#10）GDD 中详细规范。

### 状态与转换

技能系统不是有限状态机，但维护两个独立的时间维度：

**冷却维度：**
```
Ready（_cooldownTimer <= 0）  ←──── 冷却到期
     ↓ TryFireClickAttack
OnCooldown（_cooldownTimer > 0）──→ Update() 递减
```

**蓄力维度：**
```
Idle ──OnMouseDown──→ Charging ──OnMouseUp──→ Idle
                           ↓
                     OnMouseHeld 广播蓄力进度
```

**游戏状态响应：**

| GameLoopManager 状态 | 点击攻击 | 自动攻击 | 蓄力特效 |
|---------------------|---------|---------|---------|
| Playing | ✅ 正常处理 | ✅ 计时器运行 | ✅ 显示 |
| Paused | ❌ 忽略 | ❌ 计时器暂停 | ❌ 停止 |
| LevelingUp | ❌ 忽略 | ❌ 计时器暂停 | ❌ 停止 |
| Victory / Defeat | ❌ 忽略 | ❌ 停止 | ❌ 停止 |

游戏状态通过 `GameLoopManager.OnGamePaused` / `OnGameResumed` / `OnLevelUpStart` / `OnLevelUpComplete` 订阅切换，不在每帧轮询。

### 与其他系统的交互

| 方向 | 系统 | 接口 |
|------|------|------|
| 订阅 | 输入系统（#2） | `OnMouseDown`、`OnMouseHeld`、`OnMouseUp` |
| 订阅 | 游戏循环管理器（#1） | `OnGamePaused`、`OnGameResumed`、`OnLevelUpStart`、`OnLevelUpComplete` |
| 读取 | 技能数据库（#3） | `SkillData` 所有字段（只读） |
| 调用 | 点击攻击系统（#8） | `ClickAttackSystem.ExecuteAttack(AttackRequest)` |
| 订阅 | 点击攻击系统（#8） | `OnAttackExecuted(AttackResult)` — 效果被动触发 |
| 订阅 | 敌人（#10 定义） | `EnemyEvents.OnEnemyDied(EnemyData, Vector2)` — 击杀被动 |
| 被调用 | 升级系统（#11） | `AddSkill(SkillData)`、`GetActiveClickForm()`、`GetPassives()` |
| 广播 | 粒子特效系统（#12） | `OnChargeStarted(worldPos)`、`OnChargeUpdated(worldPos, ratio)` |
| 查询接口 | 游戏 HUD（#14） | `GetActiveClickForm()`、`GetCooldownProgress()` → [0, 1] |

**设计说明**：  
`EnemyEvents.OnEnemyDied` 是一个全局静态事件，由每个敌人在自身死亡时广播，而非由本系统轮询。统计系统（结算 UI #16 的前置模块）也应订阅此事件计数击杀。

## 公式

**1. 蓄力伤害（ClickForm 攻击）**

```
chargeRatio  = clamp(holdDuration / chargeTime, 0, 1)  [chargeTime > 0 时]
             = 1.0                                      [chargeTime == 0 时]

baseDamage   = skillData.damage
             × lerp(1.0, CHARGE_MULT, chargeRatio)      // 蓄力加成：默认 1.5×
             × _mods.damageMult                         // 被动伤害倍率
             × (isCrit ? CRIT_MULT : 1.0)               // 暴击：默认 2.0×
```

**2. 冷却时间**

```
actualCooldown = skillData.cooldown × _mods.cooldownMult
               ≥ skillData.cooldown × MIN_COOLDOWN_MULT  (下限 0.2，防归零)
```

**3. 被动伤害倍率（RecalculateModifiers）**

```
damageMult = 1.0
           + Σ(神力 stacks × 0.30)
           + Σ(其他 Power 被动 stacks × bonusDamagePerStack)
```

**4. 冷却缩减倍率**

```
cooldownMult = 1.0
             + Σ(神速 stacks × −0.20)    // 每层 −20%，负数
             + Σ(其他 Frequency 被动贡献)
cooldownMult = clamp(cooldownMult, 0.20, 1.0)
```

**5. 瘟疫 / 余震伤害**

```
aftershockDamage = mainAttackDamage × AFTERSHOCK_RATIO   // 默认 0.3
plagueDamage     = mainAttackDamage × PLAGUE_RATIO        // 默认 0.2
```

## 边界情况

| 情况 | 处理方式 |
|------|---------|
| 蓄力期间 GameState 变为 LevelingUp | 立即广播 `OnChargeUpdated(pos, 0)`（取消特效），`_isCharging = false`；若鼠标之后 MouseUp 事件到达，TryFire 的状态检查会拒绝执行 |
| 蓄力期间切换点击形态（理论上升级时不能点击，但防御性处理） | 新形态的 `chargeTime` 用于后续计算，蓄力起始时间保留，不重置 |
| 添加同一 Passive 超过 maxStackCount 次 | 静默拒绝，不更新层数，不重新计算（升级系统应避免提供此选项） |
| `chargeTime = 0` 但系统仍计算 chargeRatio | 特判：`chargeTime == 0` 时 `chargeRatio = 1.0`，避免除以零 |
| 余震触发时又有余震（递归） | `_processingEffectPassives` 标记阻断，余震不会二次触发余震 |
| 瘟疫链式传播杀死大量敌人（性能） | 自然有上限（每帧 OnEnemyDied 回调数量 = 该帧死亡敌人数），无需人工限制 |
| AutoAttack 在游戏暂停期间倒计时 | 订阅 GameLoopManager 事件，暂停时停止所有 `_autoTimers` 递减 |
| 狂怒减少冷却后 `_cooldownTimer < 0` | `_cooldownTimer = max(0, ...)` 确保不为负 |
| 首局游戏：AddSkill 被调用前的初始状态 | `_activeClickForm = defaultSkill`（神罚之指），由游戏循环管理器在 `OnGameStart` 时调用 `SkillSystem.Initialize(starterSkill)` |

## 调节旋钮

| 旋钮 | 默认值 | 安全范围 | 极端行为 |
|------|--------|---------|---------|
| `CHARGE_MULT` | 1.5 | 1.0 – 3.0 | 1.0 = 蓄力无意义；>2.0 = 必须蓄力否则伤害太低 |
| `CRIT_MULT` | 2.0 | 1.5 – 4.0 | <1.5 = 暴击感觉不到；>3.0 = 暴击破坏平衡 |
| `MAX_CRIT_CHANCE` | 0.8 | 0.5 – 0.95 | 1.0 = 必暴击，失去随机感 |
| `MIN_COOLDOWN_MULT` | 0.2 | 0.1 – 0.5 | <0.1 = 几乎无冷却，画面刷爆 |
| `AFTERSHOCK_RATIO` | 0.3 | 0.1 – 0.6 | >0.6 = 余震比主攻伤害高，本末倒置 |
| `AFTERSHOCK_RADIUS` | 1.5 | 0.5 – 3.0 | >3.0 接近 AOE 主攻击范围，效果重叠 |
| `PLAGUE_RATIO` | 0.2 | 0.1 – 0.5 | >0.5 = 瘟疫链式传播秒杀所有人，破坏节奏 |
| `PLAGUE_RADIUS` | 2.0 | 1.0 – 4.0 | >4.0 = 一个死亡感染全屏 |
| `RAGE_COOLDOWN_REDUCTION` | 0.5 | 0.1 – 1.0 | >1.0 = 击杀后直接清空冷却 |

## 视觉/音效需求

本系统广播以下事件供特效系统消费（本系统不产生视觉）：

- `OnChargeStarted(Vector2 worldPos)` — 蓄力特效开始
- `OnChargeUpdated(Vector2 worldPos, float chargeRatio)` — 蓄力特效强度（0→1）
- 冷却进度通过 `GetCooldownProgress()` 接口由 HUD 轮询显示（而非事件推送）

## UI 需求

**游戏 HUD 需要的查询接口：**

```csharp
SkillData GetActiveClickForm()           // 当前技能图标 + 名称
float     GetCooldownProgress()          // 0 = 可用，1 = 刚刚开冷却，返回 _cooldownTimer / actualCooldown
IReadOnlyList<PassiveEntry> GetPassives() // 被动列表（供 HUD 显示已装备技能）
```

## 验收标准

1. **点击攻击**：Playing 状态下，点击 → 蓄力 → 释放 → 正确调用 ExecuteAttack，伤害符合蓄力公式
2. **冷却忽略**：冷却期间多次点击，只在冷却结束后的下一次点击才生效，中间点击静默丢弃
3. **蓄力满伤**：满蓄力（chargeRatio = 1）伤害 = skillData.damage × 1.5 × damageMult
4. **蓄力即时**：瞬间点击（holdDuration ≈ 0）伤害 = skillData.damage × 1.0 × damageMult
5. **ClickForm 切换**：AddSkill(陨石) → `_activeClickForm` 更新，冷却重置
6. **被动倍率**：添加 2 层神力（+30%/层）→ damageMult = 1.6，攻击伤害正确放大
7. **暴击**：critChance = 1.0（测试用）→ 所有攻击均为双倍伤害
8. **余震**：添加余震被动 → OnAttackExecuted 后触发二次 AOE，不触发第三次（防递归）
9. **自动攻击**：天雷 triggerInterval = 3s → 每 3 秒调用一次 ExecuteAttack（误差 < 1 帧）
10. **暂停**：Paused 状态下，点击无效，自动攻击计时器停止
11. **LevelingUp**：LevelingUp 状态下，点击无效
12. **性能**：每帧 Update() 开销 < 0.1ms（含所有 AutoAttack 计时器）
