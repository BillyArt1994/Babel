# 主菜单 UI

> **状态**: 设计中
> **作者**: 用户 + Claude
> **最后更新**: 2026-03-29
> **实现支柱**: 支柱 4（视觉冲击力）

## 概述

主菜单是游戏启动后的第一个界面。呈现游戏标题、旧约神罚氛围的背景画面，提供"开始游戏"和"退出游戏"两个按钮。MVP 阶段无难度选择，无成就展示。点击"开始游戏"直接进入游戏场景。

## 玩家幻想

主菜单是玩家对游戏的第一印象，应在 5 秒内让玩家感受到"这是一个关于愤怒天神惩罚人类的游戏"。通过标题字体、背景画面（通天塔、乌云、雷霆元素）快速建立旧约氛围，然后以最少的摩擦引导玩家进入游戏。

## 详细设计

### 核心规则

1. 游戏启动时加载主菜单场景，显示主菜单 UI
2. "开始游戏"按钮 → 加载游戏场景，`GameLoopManager.StartGame()` 在游戏场景的 `OnEnable` 中触发
3. "退出游戏"按钮 → `Application.Quit()`
4. 无子菜单，无其他交互逻辑

### 状态与转换

| 状态 | 说明 |
|------|------|
| 主菜单场景 | 显示标题 + 两个按钮 |
| 游戏场景（过渡后） | 主菜单 UI 随场景销毁 |

### 与其他系统的交互

| 系统 | 方向 | 接口 |
|------|------|------|
| 游戏循环管理器 | 输出 → | 场景加载后，GameLoopManager 自动初始化（无需主菜单直接调用） |

## 公式

无。

## 边缘情况

1. **退出按钮在 Unity 编辑器中无效**：`Application.Quit()` 在编辑器内不起作用，属正常行为，构建版本正常
2. **场景切换期间重复点击"开始游戏"**：场景加载开始后立即禁用按钮，防止重复加载

## 依赖关系

**上游依赖：** 无（主菜单是独立场景，无运行时依赖）

**下游依赖：** 无（终端系统）

## 调优参数

无运行时可调参数。游戏标题和背景图为静态资源。

## 视觉/音频需求

- 背景：通天塔 + 乌云 + 雷霆元素的静态插画或简单动效
- 标题字体：粗壮/神圣风格，配合旧约氛围
- 可选：主菜单背景音乐（低沉、庄严，由音效系统#20 实现，MVP 可跳过）

## 验收标准

1. 游戏启动后正确显示主菜单场景
2. "开始游戏"点击后，正确加载游戏场景，游戏开始
3. "退出游戏"在构建版本中正确关闭程序
4. 快速双击"开始游戏"不导致场景重复加载

## 待解决问题

1. **难度选择**：游戏概念提到完整版包含 3 个难度等级，MVP 阶段跳过。完整版时本菜单需扩展难度选项，或添加子菜单。
2. **背景美术资源**：背景画面需要专属美术资源，MVP 阶段可用纯色背景占位。
