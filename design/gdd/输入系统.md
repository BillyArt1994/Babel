# 输入系统

> **状态**: 已完成
> **作者**: 用户 + Claude
> **最后更新**: 2026-03-28
> **实现支柱**: 支柱 1（神罚的节奏感）、支柱 5（简洁的复杂性）

## 概述

输入系统是游戏的唯一玩家输入接口，负责捕获鼠标点击事件并将其转换为游戏内的攻击指令。玩家通过鼠标左键点击屏幕上的任意位置，系统将该点击的世界坐标和时间戳广播给订阅的系统（主要是点击攻击系统和技能系统）。ESC 键由系统捕获后转发给游戏循环管理器处理暂停逻辑。

输入系统本身不执行伤害计算，也不判断点击是否命中敌人——它只负责"捕获意图，广播事件"，将原始输入与游戏逻辑彻底解耦。没有这个系统，点击攻击和技能触发将无法统一管理，且当游戏处于暂停、升级选择等非游戏状态时，无法安全屏蔽玩家输入。

## 玩家幻想

输入系统是玩家感受不到的基础设施，但它是"神之意志降临凡间"这一幻想的基石。当玩家举起手指（按住鼠标）蓄力，然后在最佳时机释放时，那一刻的爽感不来自复杂的按键组合——它来自单一、流畅的动作中蕴含的力量感。

这个系统的目标是**让输入消失**：玩家不应该感觉自己在操作 UI，而是在直接指挥神力。输入的简洁性（只有一个鼠标键）意味着玩家的注意力可以完全集中在战场态势和技能时机上，而非记忆按键。

参考：《Vampire Survivors》的自动战斗让玩家专注于位移决策；本游戏的设计则是用单一点击让玩家专注于"何时、何地释放神力"的决策。

## 详细设计

### 核心规则

输入系统的职责严格限定为：**捕获原始输入事件，查询游戏状态，决定是否广播事件**。它不执行游戏逻辑，不计算伤害，不判断命中。

**鼠标左键事件（仅在"进行中"状态下广播）：**

1. 玩家按下鼠标左键（`mousedown`）：
   - 将屏幕坐标转换为世界坐标
   - 广播 `OnMouseDown(Vector2 worldPos)`
   - 开始计时（`holdTimer = 0`）

2. 玩家持续按住鼠标左键（每帧，按住期间）：
   - 累加 `holdTimer += Time.deltaTime`
   - 持续广播 `OnMouseHeld(Vector2 worldPos, float holdDuration)`
   - `worldPos` 为当前鼠标帧位置（允许按住期间移动鼠标）

3. 玩家释放鼠标左键（`mouseup`）：
   - 广播 `OnMouseUp(Vector2 worldPos, float holdDuration)`
   - 重置 `holdTimer = 0`

**鼠标右键：** 完全忽略（预留为未来功能）。

**ESC 键（所有状态下检测，但行为不同）：**

| 当前状态 | ESC 行为 |
|---------|---------|
| 进行中 | 调用 `GameLoopManager.RequestPause()` |
| 暂停 | 调用 `GameLoopManager.RequestResume()` |
| 升级选择中 | 忽略 |
| 胜利/失败 | 忽略 |

**坐标转换规则：**
- 使用 `Camera.main.ScreenToWorldPoint(Input.mousePosition)` 将屏幕坐标转换为世界坐标
- Z 轴固定为 0（2D 游戏）

### 状态与转换

输入系统**不维护自身状态**——它通过查询 `GameLoopManager.GetCurrentState()` 决定行为。这避免了状态同步问题。

```
每帧执行顺序：
1. 读取 GameLoopManager.GetCurrentState()
2. 如果 state == Playing：
   a. 检测鼠标左键输入，广播相应事件
   b. 检测 ESC 键，转发给 GameLoopManager
3. 如果 state == Paused：
   a. 检测 ESC 键，转发给 GameLoopManager
   b. 其他输入忽略
4. 如果 state == LevelingUp / Victory / Defeat：
   a. 所有输入忽略（UI 由 Unity EventSystem 处理）
```

### 与其他系统的交互

**输出接口（输入系统广播，下游系统订阅）：**

| 事件 | 参数 | 订阅方 |
|-----|------|--------|
| `OnMouseDown` | `Vector2 worldPos` | 技能系统、点击攻击系统 |
| `OnMouseHeld` | `Vector2 worldPos, float holdDuration` | 技能系统（蓄力特效驱动） |
| `OnMouseUp` | `Vector2 worldPos, float holdDuration` | 技能系统（技能释放判断） |

**输入依赖（输入系统查询）：**

| 系统 | 查询方法 | 用途 |
|-----|---------|------|
| 游戏循环管理器 | `GetCurrentState()` | 判断是否广播事件 |
| 游戏循环管理器 | `RequestPause()` / `RequestResume()` | 转发 ESC 操作 |

## 公式

1. **世界坐标转换**
   ```
   Vector3 screenPos = Input.mousePosition
   screenPos.z = Camera.main.nearClipPlane
   worldPos = Camera.main.ScreenToWorldPoint(screenPos)
   worldPos.z = 0  // 强制 Z = 0（2D）
   ```

2. **按住时长计算**
   ```
   holdDuration += Time.deltaTime  // 每帧，仅在 isMouseHeld == true 时累加
   ```
   - 初始值：`holdDuration = 0`（每次 mousedown 时重置）
   - 最大值：无上限（由技能系统决定蓄力上限）
   - 单位：秒

3. **无其他公式** — 这个系统是纯事件路由，没有伤害、冷却或概率计算。

## 边缘情况

1. **鼠标按住期间游戏暂停**
   - 情况：玩家按住鼠标蓄力，然后按 ESC 暂停
   - 处理：立即广播 `OnMouseUp(worldPos, holdDuration)` 强制结束按住状态，然后转发暂停请求，重置 `holdTimer = 0`
   - 原因：避免恢复游戏后出现"幽灵按住"状态（鼠标实际已放开但系统认为仍在按住）

2. **鼠标移出窗口期间仍按住**
   - 情况：玩家按住鼠标移出游戏窗口，然后在窗口外释放，再移回窗口
   - 处理：在 `OnApplicationFocus(false)` 时强制发送 `OnMouseUp`，重置状态
   - 原因：Unity 在窗口失焦时不会收到 mouseup 事件

3. **游戏结束时仍有按住事件在进行中**
   - 情况：胜利/失败判定的同一帧，玩家正在按住鼠标
   - 处理：立即发送 `OnMouseUp`，然后屏蔽后续所有事件
   - 原因：防止技能系统在游戏结束后还收到 `OnMouseHeld` 事件

4. **摄像机为空**
   - 情况：`Camera.main` 为 null（场景切换中）
   - 处理：跳过该帧的输入处理，不广播任何事件
   - 原因：防止空引用异常

5. **极快连续点击（误触）**
   - 情况：玩家每帧按下/释放鼠标（帧率 > 手速，但仍可能触发多次 `OnMouseDown`）
   - 处理：无需特殊处理——技能系统负责管理冷却，输入系统如实广播所有点击
   - 原因：输入系统不应做游戏逻辑判断

## 依赖关系

**上游依赖（此系统依赖的系统）：**

| 系统 | 依赖类型 | 接口 |
|-----|---------|------|
| 游戏循环管理器 | 硬依赖 | `GetCurrentState()` — 查询游戏状态以决定是否广播事件 |
| 游戏循环管理器 | 硬依赖 | `RequestPause()` / `RequestResume()` — 转发 ESC 操作 |

**下游依赖（依赖此系统的系统）：**

| 系统 | 依赖类型 | 订阅的接口 |
|-----|---------|----------|
| 点击攻击系统 | 硬依赖 | `OnMouseDown` — 每次点击触发单次攻击判断 |
| 技能系统 | 硬依赖 | `OnMouseDown` / `OnMouseHeld` / `OnMouseUp` — 技能释放、蓄力驱动 |

## 调优参数

| 参数名 | 默认值 | 安全范围 | 说明 |
|--------|--------|---------|------|
| `INPUT_ENABLED_STATES` | `[Playing]` | — | 允许广播攻击事件的游戏状态列表，硬编码，不可运行时调整 |

**无其他调优参数：**
- 坐标转换逻辑由 Unity Camera API 决定，不需要调整
- 按住时长由技能系统定义，不属于输入系统的调优范围
- 冷却/速率限制不属于输入层职责

**不可调参数（硬编码）：**
- 事件名称（`OnMouseDown` / `OnMouseHeld` / `OnMouseUp`）
- 鼠标按键映射（左键 = 攻击，右键 = 忽略）
- Z 轴世界坐标固定为 0

## 验收标准

**功能测试：**
1. 游戏进行中，鼠标左键点击时，`OnMouseDown` 事件正确携带世界坐标广播
2. 游戏进行中，持续按住鼠标，每帧收到 `OnMouseHeld` 事件，`holdDuration` 单调递增
3. 游戏进行中，释放鼠标左键，`OnMouseUp` 事件携带正确的 `holdDuration` 广播
4. 游戏暂停时，点击鼠标不触发任何攻击事件
5. 升级选择中，点击鼠标不触发任何攻击事件（UI 点击仍正常工作）
6. 游戏进行中按 ESC，游戏循环管理器收到暂停请求
7. 游戏暂停时按 ESC，游戏循环管理器收到恢复请求
8. 升级选择中按 ESC，无任何响应

**边缘情况测试：**
9. 按住鼠标期间按 ESC 暂停，恢复后 `holdTimer` 已重置，无幽灵按住状态
10. 按住鼠标移出窗口再移回，系统不认为鼠标仍在按住状态
11. `Camera.main` 为 null 时，不抛出异常，该帧输入静默跳过

**性能标准：**
12. 每帧输入处理耗时不超过 0.05ms
13. 不产生 GC 分配（使用 C# 事件系统，避免每帧 `new` 对象）
