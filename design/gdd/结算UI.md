# 结算 UI

> **状态**: 设计中
> **作者**: 用户 + Claude
> **最后更新**: 2026-03-29
> **实现支柱**: 支柱 2（压力递增的绝望感）、支柱 4（视觉冲击力）

## 概述

结算 UI 在游戏结束时以全屏覆盖呈现，以宏大叙事文本渲染结果：胜利时展示天神威严降临的神圣感，失败时展示人类僭越成功的末日氛围。唯一的数值数据是击杀数，作为玩家"本局战果"的见证。界面不显示塔完整度数据，保持对游戏世界状态的叙事性模糊。底部提供"再战一局"和"返回主菜单"按钮。

## 玩家幻想

失败时，玩家看到"通天塔已建成——人类抵达天庭"这样的文本，感受到失败的"重量"而非干燥的数字；同时击杀数告诉他"我消灭了 847 个人类"，给失败赋予一种英雄悲剧感，而非挫败感。胜利时，"天神的震怒平息，巴别塔化为尘埃"的文本配合本局击杀数，让玩家感到自己真的扮演了那个愤怒的神。结算界面是游戏体验的"落幕"，应该有仪式感。

## 详细设计

### 核心规则

1. 监听 `GameLoopManager.OnVictory` 和 `OnDefeat` 事件
2. 事件触发时，全屏覆盖场景（HUD 也随之隐藏），显示结算界面
3. 根据事件类型填充内容：
   - 胜利：主标题 = 胜利文本（见调优参数）；副文本 = 胜利氛围描述
   - 失败：主标题 = 失败文本；副文本 = 失败氛围描述
4. 击杀数从 StatsTracker 读取，显示"消灭了 N 名人类"
5. "再战一局"按钮：调用 `GameLoopManager.RestartGame()`
6. "返回主菜单"按钮：调用 `GameLoopManager.ReturnToMainMenu()`（MVP 阶段若无主菜单，直接重新加载场景）

### 状态与转换

| 状态 | 显示 | 进入条件 | 退出条件 |
|------|------|---------|---------|
| 隐藏 | 不可见 | 初始 / 游戏进行中 | `OnVictory` 或 `OnDefeat` |
| 显示中（胜利） | 胜利全屏界面 | `OnVictory` | 点击"再战一局"或"返回主菜单" |
| 显示中（失败） | 失败全屏界面 | `OnDefeat` | 点击"再战一局"或"返回主菜单" |

### 与其他系统的交互

| 系统 | 方向 | 接口 |
|------|------|------|
| 游戏循环管理器 | 输入 ← | 监听 `OnVictory`、`OnDefeat`；调用 `RestartGame()`、`ReturnToMainMenu()` |
| StatsTracker | 输入 ← | `GetKillCount()` — 读取本局击杀数 |

## 公式

无数值计算。击杀数文本：
```
display = string.Format("消灭了 {0} 名人类", StatsTracker.GetKillCount())
```

## 边缘情况

1. **同帧同时触发胜利和失败**：游戏循环管理器 GDD 规定"失败优先"，结算 UI 只响应第一个事件
2. **界面显示期间点击两次按钮**：第一次点击后按钮立即禁用，防止重复调用
3. **StatsTracker 返回 0**：正常展示"消灭了 0 名人类"，不作特殊处理
4. **主菜单 UI 尚未实现（MVP 阶段）**：`ReturnToMainMenu()` 临时实现为重新加载当前场景，行为等同"再战一局"

## 依赖关系

**上游依赖：**

| 系统 | 依赖类型 | 接口 |
|-----|---------|------|
| 游戏循环管理器 | 硬依赖 | `OnVictory`、`OnDefeat` 事件；`RestartGame()`、`ReturnToMainMenu()` |
| StatsTracker | 硬依赖 | `GetKillCount()` |

**下游依赖（依赖此系统的系统）：**

| 系统 | 说明 |
|------|------|
| 解锁系统（#18） | 监听结算 UI 显示事件（或直接监听 GameLoopManager 事件）检查成就 |
| 排行榜系统（#19） | 监听结算 UI 显示事件（或直接监听 GameLoopManager 事件）记录分数 |

> **注：** 解锁系统和排行榜系统可直接监听 `GameLoopManager.OnVictory` / `OnDefeat`，无需通过结算 UI 路由。

## 调优参数

| 参数名 | 默认值 | 说明 |
|--------|--------|------|
| `VICTORY_TITLE` | "天神的震怒平息，巴别塔化为尘埃" | 胜利主标题文本 |
| `VICTORY_SUBTITLE` | "人类的僭越之心，终被镇压于历史长河" | 胜利副标题氛围文本 |
| `DEFEAT_TITLE` | "通天塔已建成——人类抵达了天庭" | 失败主标题文本 |
| `DEFEAT_SUBTITLE` | "审判日已降临，神的宝座第一次被动摇" | 失败副标题氛围文本 |

> 文本为占位草稿，正式版由创意总监/文案确认。

## 视觉/音频需求

- **胜利界面**：金色/神圣光效背景，庄严感
- **失败界面**：暗红/烟雾背景，末日感
- 可选：胜利时播放短暂的神圣音效；失败时播放沉重的钟声（由音效系统#20 实现，MVP 可跳过）
- 界面以淡入方式出现（约 0.5s），覆盖游戏场景

## UI 需求

**布局（概念）：**
```
┌────────────────────────────────────────┐
│                                        │
│    [主标题文本（大字）]                  │
│                                        │
│    [氛围副文本（中字）]                  │
│                                        │
│    消灭了 N 名人类                      │
│                                        │
│    [再战一局]    [返回主菜单]            │
└────────────────────────────────────────┘
```

- Canvas：Screen Space - Overlay，最高层级
- 背景：全屏纹理/颜色（胜利/失败各一套）
- 按钮：等宽，底部居中排列

## 验收标准

1. 倒计时归零且塔未完成时，显示胜利界面，主标题为胜利文本
2. 塔完成度 100% 时，显示失败界面，主标题为失败文本
3. 击杀数正确从 StatsTracker 读取并显示
4. "再战一局"触发游戏重置，重新从零开始
5. "返回主菜单"触发场景切换（MVP 阶段等同重置）
6. 快速点击按钮两次不触发重复操作
7. 结算界面显示后，游戏 HUD 不可见
