# 点击攻击系统

> **状态：** 已完成
> **作者：** 独立开发者 + AI
> **最后更新：** 2026-03-28
> **实现支柱：** 支柱1（神罚节奏感）、支柱4（视觉冲击力）

## 概述

点击攻击系统是薄执行层。对外暴露唯一入口 `ExecuteAttack(AttackRequest)`，执行物理碰撞检测，调用命中敌人的 `TakeDamage()`，广播 `OnAttackExecuted(AttackResult)` 事件供粒子特效和音效系统消费。

本系统不管理冷却，不决定使用哪个技能，不产生视觉效果——完全无状态。技能系统（#9）是本系统的唯一调用方。

## 玩家幻想

服务支柱1（神罚节奏感）+ 支柱4（视觉冲击力）。

点击即见效，命中有即时反馈，锁链跳跃产生层次感。玩家感受到的是"神之手直接触碰凡人"的即时性——不是发射物，不是延迟，而是指哪打哪的全能感。视觉和音效冲击由下游系统（#12、#20）基于本系统广播的事件实现。

## 详细设计

### 核心数据结构

```csharp
public enum AttackType { Single, AOE, Pierce, Chain }

public struct AttackRequest
{
    public Vector2    worldPos;        // 点击的世界坐标
    public AttackType attackType;
    public float      damage;          // 基础伤害（由技能系统传入）
    public float      radius;          // AOE 范围（Single/Pierce 忽略）
    public int        chainCount;      // 锁链跳跃次数（Chain 专用）
    public float      chainRadius;     // 锁链寻找下一目标的搜索半径
    public float      chainDecay;      // 锁链每跳伤害系数（默认 0.8）
    public float      slowPercent;     // 减速百分比（0 = 无减速）
    public float      slowDuration;    // 减速持续时间（秒）
}

public struct EnemyHitInfo
{
    public IDamageable target;
    public float       damageDealt;    // 实际扣除伤害
    public Vector2     hitPosition;
    public int         chainIndex;     // 0=直接命中，1=第一跳，以此类推
}

public struct AttackResult
{
    public Vector2        origin;
    public AttackType     attackType;
    public EnemyHitInfo[] hits;        // 空数组表示未命中任何目标
}

public interface IDamageable
{
    void    TakeDamage(float damage, float slowPercent, float slowDuration);
    bool    IsAlive   { get; }
    Vector2 Position  { get; }
}
```

### 核心规则

1. 技能系统调用 `ExecuteAttack(AttackRequest request)`
2. 根据 `request.attackType` 分发到对应检测逻辑（见下表）
3. 用 `HashSet<IDamageable>` 去重，确保同一敌人在单次攻击中只被命中一次
4. 对每个命中目标调用 `target.TakeDamage(damageDealt, slowPercent, slowDuration)`
5. 若命中列表为空（无目标），**不广播**事件，静默返回
6. 否则广播 `static event Action<AttackResult> OnAttackExecuted`
7. 本系统**不保存任何状态**，每次调用完全独立

### 4 种攻击类型检测

| 类型 | 对应技能（示例） | 检测方式 | 命中目标 |
|------|---------------|---------|---------|
| **Single** | 神罚之指 | `Physics2D.OverlapPoint(worldPos)` | 仅点击到的那个敌人（精准碰撞体，容错靠碰撞体大小调整） |
| **AOE** | 天降陨石 | `Physics2D.OverlapCircleAll(worldPos, radius)` | 范围内全部（上限 `maxAOETargets`，默认 64） |
| **Pierce** | 圣光洪流 | `Physics2D.OverlapBoxAll(垂直条带)` | 点击 X 坐标的垂直条带内全部 |
| **Chain** | 神雷锁链 | Single 取首目标 → BFS 最近未命中活体 × `chainCount` 次 | 首目标 + 最多 N 跳 |

**Pierce 垂直条带定义**：
- 中心 X = `worldPos.x`，中心 Y = 视口中心
- 宽度 = `pierceLineHalfWidth × 2`（默认 1.0 单位）
- 高度 = 视口高度 + 4 单位缓冲（上下各 2）

**Chain BFS 规则**：
- 每跳在上一命中目标周围 `chainRadius` 范围内找最近的**未命中**活体
- 若没有可用目标则提前终止，以实际跳跃次数记录结果

### 状态与转换

本系统无状态机，无持久化数据。`ExecuteAttack()` 是纯函数式调用（读取世界状态，写入伤害，广播事件），不保留执行结果。

### 与其他系统的交互

| 方向 | 系统 | 接口 |
|------|------|------|
| **被调用** | 技能系统（#9） | `ClickAttackSystem.ExecuteAttack(AttackRequest)` |
| **调用** | 敌人单位（通过接口） | `IDamageable.TakeDamage(damage, slow%, slowDur)` |
| **广播** | 粒子特效系统（#12） | 订阅 `OnAttackExecuted(AttackResult)` |
| **广播** | 音效系统（#20） | 订阅 `OnAttackExecuted(AttackResult)` |

**架构说明**：本系统在依赖图中列于输入系统（#2）之后，实际上输入事件由技能系统（#9）消费后才调用本系统。这是「方案A——薄执行层」架构选择的设计后果，技能系统 GDD 中需明确记录此调用路径。

## 公式

| 攻击类型 | 伤害公式 |
|---------|---------|
| Single | `damageDealt = request.damage` |
| AOE | `damageDealt = request.damage`（对范围内每个目标相同） |
| Pierce | `damageDealt = request.damage`（对条带内每个目标相同） |
| Chain | `damageDealt = request.damage × (chainDecay ^ chainIndex)` |

**Chain 衰减示例**（chainDecay = 0.8，damage = 100）：
- 第 0 跳（直接命中）：100 × 0.8⁰ = **100**
- 第 1 跳：100 × 0.8¹ = **80**
- 第 2 跳：100 × 0.8² = **64**
- 第 3 跳：100 × 0.8³ ≈ **51**

## 边界情况

| 情况 | 处理方式 |
|------|---------|
| Single / Pierce 点击空白处（无敌人） | 构建空 hits[] 的 AttackResult，**不广播**事件，无声返回 |
| Chain 途中目标在同帧死亡 | 跳跃前检查 `IsAlive`，跳过死亡目标，继续寻找下一活体 |
| AOE 范围内超过 `maxAOETargets` 个敌人 | 取 Physics2D 返回的前 N 个，记录 Warning 日志（设计上后期应不触发） |
| Pierce 检测到同一敌人两次（宽碰撞体） | HashSet 去重，每个目标只调用一次 TakeDamage |
| 游戏处于 LevelingUp / Paused 状态时收到调用 | 技能系统负责拦截，**不应调用** ExecuteAttack；本系统不做状态检查（单一职责） |
| `chainDecay = 1.0f` | 正常工作，1.0⁰ = 1.0，全跳等伤，无分支 |
| 首目标死于 TakeDamage 调用后 Chain 寻找下一跳 | 正常，Chain BFS 从死亡目标的 Position 出发查找周围活体 |

## 依赖关系

| 类型 | 系统 | 性质 |
|------|------|------|
| 正式依赖 | 输入系统（#2） | 架构层级依赖（通过技能系统中转） |
| 被调用 | 技能系统（#9） | 硬依赖（无#9则无人调用本系统） |
| 被订阅 | 粒子特效系统（#12） | 软依赖（无#12也可运行，只是无视觉） |
| 被订阅 | 音效系统（#20） | 软依赖（无#20也可运行，只是无音效） |
| 接口依赖 | IDamageable（敌人实现） | 编译期接口，运行时由敌人AI系统（#10）实现 |

## 调节旋钮

| 旋钮 | 默认值 | 安全范围 | 极端行为 |
|------|--------|---------|---------|
| `chainDecay` | 0.8 | 0.5 – 1.0 | <0.5：后几跳几乎 0 伤害；=1.0：全跳等伤 |
| `defaultChainRadius` | 3.0 单位 | 1.0 – 6.0 | <1.0：锁链几乎只打第一目标；>6：跨半屏跳跃 |
| `pierceLineHalfWidth` | 0.5 单位 | 0.1 – 1.5 | <0.1：接近精准线；>1.5：效果接近小范围 AOE |
| `maxAOETargets` | 64 | 16 – 128 | <16：大范围 AOE 被截断；>128：可能有 Physics2D 性能影响 |

所有旋钮应作为 `[SerializeField]` 暴露在 Inspector，支持运行时调整。

## 视觉/音效需求

本系统不直接负责视觉或音效，通过 `OnAttackExecuted(AttackResult)` 事件向下游提供数据：

- 粒子特效系统（#12）使用 `AttackResult.origin`、`attackType`、`hits[]` 的位置决定特效类型和触发位置
- 音效系统（#20）使用 `attackType` 和 `hits.Length` 决定音效变体（命中多目标可触发更重的音效）

**事件数据完整性要求**：`hits[].hitPosition` 必须是敌人的实际世界坐标（非点击坐标），以便粒子效果准确附着在命中位置。

## UI 需求

无直接 UI 需求。伤害数字（浮动文本）若需要实现，应由独立的伤害数字系统订阅 `OnAttackExecuted` 事件，而非由本系统负责。

## 验收标准

1. **Single 命中**：点击命中敌人碰撞体 → 敌人扣血 → OnAttackExecuted 广播，hits.Length == 1
2. **Single 未命中**：点击空白处 → 无事件广播，无报错，无 GC
3. **AOE**：Physics2D.OverlapCircleAll 范围内所有敌人各自扣等值伤害
4. **Pierce**：垂直条带内所有敌人扣等值伤害，HashSet 确保无重复
5. **Chain 伤害衰减**：首跳 100%，第二跳 80%，第三跳 64%（±0.5% 浮点误差可接受）
6. **Chain 不重复**：同一敌人在单次 Chain 中只被命中一次
7. **Chain 目标死亡**：途中目标死亡时跳过，Chain 正常继续到下一活体
8. **性能**：1000 次 ExecuteAttack 调用，零 GC 分配（全 struct 值类型，避免装箱）
9. **状态检查**：切换到 LevelingUp 状态后，技能系统停止调用，本系统不自行检查状态
