# 敌人数据库

> **状态**: 已完成
> **作者**: 用户 + Claude
> **最后更新**: 2026-03-28
> **实现支柱**: 支柱 2（压力递增的绝望感）、支柱 5（简洁的复杂性）

## 概述

敌人数据库是游戏所有人类单位类型的配置仓库，以 Unity ScriptableObject 形式存在。每种敌人是一个独立的 `EnemyData` 资产，存储该类型的静态属性（血量、速度、建造贡献值）、出场时间，以及特殊能力配置。系统本身不执行 AI 逻辑——它只是数据。敌人生成系统读取出场时间来决定何时生成哪种单位；对象池系统根据 Prefab 引用创建单位实例。

## 玩家幻想

敌人数据库对玩家不可见，但它是"压力递增"体验的数据基础。Worker 血量低、Elite 血量是它的 3-5 倍、Priest 会治疗周围人类——这些精心设定的数值差异让玩家在游戏中段感受到节节升高的威胁。数据库的价值在于：设计师可以在不改代码的情况下，通过调整 Inspector 数值来迭代难度曲线。

## 详细设计

### 核心规则

每种敌人以一个 `EnemyData` ScriptableObject 资产存在，保存在 `Assets/Data/Enemies/` 目录下。

**`EnemyData` 字段定义：**

#### 元数据字段
| 字段名 | 类型 | 说明 |
|--------|------|------|
| `enemyId` | `string` | 唯一标识符（如 `"worker"`、`"priest"`），用于代码引用 |
| `enemyName` | `string` | 显示名称（如"普通人类"） |
| `prefab` | `GameObject` | 此类型的 Unity Prefab，由对象池系统用于创建实例 |

#### 基础属性字段
| 字段名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `baseHealth` | `float` | `30` | 基础血量 |
| `moveSpeed` | `float` | `2.0` | 移动速度（Unity 单位/秒） |
| `buildContribution` | `float` | `1.0` | 每次到达塔基时增加的完整度百分比 |
| `spawnStartTime` | `float` | `0` | 最早出场时间（秒），0 表示游戏开始即可出现 |
| `spawnWeight` | `float` | `1.0` | 生成池中的权重，越高越频繁生成 |

#### 特殊能力字段（Priest 专用）
| 字段名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `healRadius` | `float` | `0` | 治疗范围半径（Unity 单位），0 表示无治疗 |
| `healPerSecond` | `float` | `0` | 每秒治疗量，0 表示无治疗 |

#### 特殊能力字段（Zealot 专用）
| 字段名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `deathExplosionRadius` | `float` | `0` | 死亡爆炸范围（Unity 单位），0 表示无爆炸 |
| `deathExplosionForce` | `float` | `0` | 爆炸推力（向塔方向推进周围人类），0 表示无推进 |

**初始 5 种敌人参考数值：**

| 敌人类型 | `baseHealth` | `moveSpeed` | `buildContribution` | `spawnStartTime` | 特殊能力 |
|---------|------------|------------|-------------------|-----------------|--------|
| Worker | 30 | 2.0 | 1.0% | 0s | 无 |
| Elite | 120 | 3.0 | 1.0% | 120s | 无 |
| Priest | 60 | 1.5 | 1.0% | 300s | healRadius:3, healPerSecond:10 |
| Engineer | 60 | 2.0 | 2.0% | 300s | 无（buildContribution=2 即为能力） |
| Zealot | 20 | 4.5 | 1.0% | 480s | explosionRadius:2, explosionForce:5 |

### 状态与转换

数据库系统无运行时状态——`EnemyData` 资产在游戏启动时加载到内存，全生命周期只读。无状态转换。

### 与其他系统的交互

**提供给其他系统的数据（只读）：**

| 消费方 | 读取的字段 | 用途 |
|--------|-----------|------|
| 敌人生成系统 | `spawnStartTime`、`spawnWeight`、`prefab` | 决定何时生成哪种单位 |
| 敌人 AI 系统 | `moveSpeed`、`healRadius`、`healPerSecond`、`deathExplosionRadius`、`deathExplosionForce` | 驱动单位行为和特殊能力 |
| 塔建造系统 | `buildContribution` | 计算到达单位对当前层完整度的贡献 |
| 点击攻击系统 | `baseHealth` | 计算击杀所需伤害 |

**输入依赖：** 无（数据库不依赖任何运行时系统）。

## 公式

技能数据库本身不执行计算，以下是各字段的预期使用方式，供下游系统参考：

1. **生成资格检查（敌人生成系统执行）**
   ```
   isEligible = GameLoopManager.GetElapsedTime() >= enemyData.spawnStartTime
   ```

2. **加权随机生成（敌人生成系统执行）**
   ```
   P(enemy) = enemyData.spawnWeight / Σ(weight of all eligible enemies)
   ```

3. **完整度更新（塔建造系统执行）**
   ```
   layerCompletion += enemyData.buildContribution  // 每次单位到达塔基时
   ```

## 边缘情况

1. **`enemyId` 重复**
   - 处理：`OnValidate()` 扫描所有 EnemyData 资产，检测重复 ID，Inspector 显示警告

2. **`prefab = null`**
   - 处理：`OnValidate()` 报错；此敌人不进入生成池，防止对象池创建 null 实例崩溃

3. **`spawnWeight = 0`**
   - 处理：敌人生成系统跳过该类型，允许此行为（可临时禁用某类型）

4. **`buildContribution = 0`**
   - 处理：允许（该单位到达塔基不贡献进度，逻辑上合法，可用于未来设计纯战斗单位）

5. **`healRadius > 0` 但场景中无其他人类**
   - 处理：Priest 治疗技能空转，无副作用，正常

6. **`deathExplosionForce` 过大推力**
   - 处理：敌人 AI 系统负责钳制推进后的位置，防止单位飞出屏幕边界；数据库层不处理

## 依赖关系

**上游依赖（此系统依赖的系统）：** 无（纯数据层）

**下游依赖（依赖此系统的系统）：**

| 系统 | 依赖类型 | 读取的字段 |
|-----|---------|----------|
| 敌人生成系统 | 硬依赖 | `prefab`、`spawnStartTime`、`spawnWeight` |
| 敌人 AI 系统 | 硬依赖 | `moveSpeed`、`healRadius`、`healPerSecond`、`deathExplosionRadius`、`deathExplosionForce` |
| 塔建造系统 | 软依赖 | `buildContribution` |
| 点击攻击系统 | 软依赖 | `baseHealth` |

## 调优参数

| 字段名 | 推荐范围 | 极端行为 |
|--------|---------|---------|
| `baseHealth` | [10, 500] | < 10 一击必杀破坏技能意义；> 500 精英单位极难处理 |
| `moveSpeed` | [0.5, 6.0] | < 0.5 速度感消失；> 6 玩家无法反应，破坏支柱 1 节奏感 |
| `buildContribution` | [0.5, 5.0] | > 5 单个单位贡献过大，几次漏放即可导致失败 |
| `spawnStartTime` | [0, 840] | > 840 只剩最后 1 分钟出现，无实质影响 |
| `healPerSecond` | [0, 30] | > 30 治疗速度超过玩家输出，Priest 变得无法处理 |
| `deathExplosionForce` | [0, 10] | > 10 推力过大，单位可能飞出地图边界 |

## 验收标准

1. 每个 EnemyData 资产可在 Unity Inspector 中独立创建和配置
2. 重复 `enemyId` 在 Inspector 中显示警告（`OnValidate()`）
3. `prefab = null` 时，该敌人不进入生成池，游戏不崩溃
4. `spawnWeight = 0` 时，该敌人永不被生成
5. 敌人生成系统可正确根据 `spawnStartTime` 过滤生成资格
6. 敌人生成系统可正确使用 `spawnWeight` 进行加权随机生成
7. 修改任意 EnemyData 字段后，下次游戏启动时生效（无需重编译代码）
