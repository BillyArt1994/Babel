# 粒子特效系统

> **状态：** 已完成
> **作者：** 独立开发者 + AI
> **最后更新：** 2026-03-28
> **实现支柱：** 支柱4（视觉冲击力）

## 概述

粒子特效系统是纯消费者（Pure Consumer）：订阅来自点击攻击系统（#8）和技能系统（#9）的事件，在正确位置播放对应的 Unity ParticleSystem 预制件。本系统不持有游戏逻辑，不产生事件，不影响任何数值计算。

所有特效预制件通过对象池系统（#5）管理，播放完毕自动归还池中。五大效果类别覆盖所有视觉反馈需求：主动点击、被动自动攻击、荷电弧线、死亡爆碎、蓄力特效。

## 玩家幻想

服务支柱4（视觉冲击力）。

每次神力释放都应有"神明降怒"的重量感：陨石砸下时地面震裂，圣光洪流穿透人群，锁链雷电在凡人间跳跃。被动特效（自动天雷、神火）则作为持续存在的"神明注视"，让屏幕始终有低频震慑感。玩家不需要思考特效——他们只需要感受到每次点击都"值"。

## 详细设计

### 核心规则

1. 本系统在 `OnEnable()` 订阅所有触发事件，在 `OnDisable()` 取消订阅
2. 所有特效实例通过 `ObjectPool<ParticleSystem>` 获取（按 EffectKey 分池）
3. 每个特效预制件挂载 `AutoReturnToPool` 组件，`ParticleSystem.isPlaying == false` 时自动归还
4. 主动点击特效分两层：**Origin（释放点）** 和 **Impact（命中点）**，各自独立播放
5. 当 `isPassiveAttack == true` 时，跳过 Origin 层，Impact 改用轻量版变体（小尺寸、短时长）
6. 荷电弧线（Chain）使用 `LineRenderer + 动画纹理` 而非粒子，单独管理生命周期
7. 蓄力特效为单例：同时只存在一个蓄力实例，`OnChargeStarted` 激活，释放/取消时回收

---

### 效果分类与触发

#### 1. 主动点击特效（Active Click Effects）

**触发条件**：`OnAttackExecuted(AttackResult)` 且 `result.hits.Length > 0` 且 `isPassiveAttack == false`

**双层结构**：

| 层级 | 播放位置 | 每种 AttackType 使用的预制件 |
|------|---------|---------------------------|
| Origin 层 | `result.origin` | 见下表 |
| Impact 层 | 每个 `hit.hitPosition` | 见下表 |

**AttackType → 预制件映射**：

| AttackType | Origin 效果描述 | Impact 效果描述 |
|-----------|----------------|----------------|
| **Single** | 指尖发光、神力聚集（短暂） | 敌人身上电弧火花爆发 |
| **AOE** | 天空裂缝、陨石坠落轨迹 | 落点圆形冲击波 + 尘土 |
| **Pierce** | 天空出现竖直光柱（沿 origin.x） | 穿透每个命中敌人的光环闪烁 |
| **Chain** | 首个目标处蓄电光环 | 每跳目标处小型电弧爆发 |

**Chain 额外**：每对相邻命中目标 `hits[i] → hits[i+1]` 播放一条荷电弧线（见下文第3类）。

---

#### 2. 被动自动攻击特效（Passive AutoAttack Effects）

**触发条件**：`OnAttackExecuted(AttackResult)` 且 `isPassiveAttack == true`

- **跳过** Origin 层（被动攻击无"释放源"需要视觉化）
- Impact 层使用**轻量变体**（prefab 名加 `_Light` 后缀），参数：尺寸 × 0.6，时长 × 0.5，粒子数 × 0.4
- 目的：主动点击保持视觉主角地位，被动攻击作为"背景持续震慑"

**天雷特殊处理**：天雷自动攻击（Single）额外在命中位置播放一个短促下落光柱（≈ 0.3 秒），区别于玩家的精准神罚之指效果

---

#### 3. 荷电弧线特效（Chain Arc Effects）

**触发条件**：主动点击且 AttackType == Chain，`hits.Length >= 2`

**实现**：
```
对每对 (hits[i], hits[i+1])：
    从 ChainArcPool 获取一个 LineRenderer
    设置 startPos = hits[i].hitPosition
    设置 endPos   = hits[i+1].hitPosition
    播放电弧动画（纹理 UV 偏移 + 噪声扰动）
    ARC_DURATION 秒后归还池
```

**视觉描述**：不规则锯齿电弧线，蓝白色，宽度约 0.1 单位，持续 `ARC_DURATION`（默认 0.25 秒）后消失，伴随小型粒子闪光

---

#### 4. 死亡爆碎特效（Death Effects）

**触发条件**：`EnemyEvents.OnEnemyDied(EnemyData data, Vector2 pos)`

- MVP 阶段：所有敌人类型共用同一套死亡特效预制件
- 播放位置：`pos`（敌人死亡时世界坐标）
- 效果描述：碎石粒子散射 + 人形轮廓短暂残影消散 + 小型光晕爆炸

---

#### 5. 蓄力特效（Charge Effects）

**触发条件**：`SkillSystem.OnChargeStarted(Vector2 pos)` / `OnChargeUpdated(Vector2 pos, float ratio)`

**生命周期**：
```
OnChargeStarted(pos)
    → 从 ChargePool 获取实例，激活于 pos
    → 保存引用 _activeChargeFX

OnChargeUpdated(pos, ratio)
    → 更新 _activeChargeFX 的播放速率 = lerp(MIN_CHARGE_RATE, MAX_CHARGE_RATE, ratio)
    → ratio = 1.0 时：启动"满蓄力脉冲"循环动画
    → ratio = 0（取消）时：立即归还池，_activeChargeFX = null

OnAttackExecuted（任意类型）
    → 若 _activeChargeFX != null：立即归还池
    → Origin 效果接管视觉（自然衔接）
```

**效果描述**：玩家指针周围发光能量聚集，随 chargeRatio 增大变亮变大，满蓄力时闪烁提示

### 与其他系统的交互

| 方向 | 系统 | 接口 |
|------|------|------|
| 订阅 | 点击攻击系统（#8） | `ClickAttackSystem.OnAttackExecuted(AttackResult)` |
| 订阅 | 技能系统（#9） | `SkillSystem.OnChargeStarted(Vector2)` |
| 订阅 | 技能系统（#9） | `SkillSystem.OnChargeUpdated(Vector2, float)` |
| 订阅 | 敌人（#10 定义） | `EnemyEvents.OnEnemyDied(EnemyData, Vector2)` |
| 使用 | 对象池系统（#5） | `ObjectPool<ParticleSystem>.Get(key)` / `.Return(instance)` |
| 读取 | AttackResult 结构 | `attackType`, `origin`, `hits[].hitPosition`, `isPassiveAttack` |

## 公式

**蓄力特效播放速率**
```
playbackRate = lerp(MIN_CHARGE_RATE, MAX_CHARGE_RATE, chargeRatio)
MIN_CHARGE_RATE = 0.3   // 刚开始蓄力时的基础播放速率
MAX_CHARGE_RATE = 2.5   // 满蓄力时的播放速率（视觉上更紧迫）
```

**轻量变体参数**
```
lightScale    = basePrefab.scale × 0.6
lightDuration = basePrefab.duration × 0.5
lightCount    = basePrefab.particleCount × 0.4
```

**荷电弧线生命周期**
```
ARC_DURATION = 0.25 秒（与 Chain Impact 特效时长同步，同时消失）
```

**对象池容量参考**

| 效果类别 | 推荐池容量 | 依据 |
|---------|----------|------|
| SingleOriginFX | 5 | 玩家点击频率有冷却限制 |
| AOE_OriginFX | 5 | 同上 |
| Pierce_OriginFX | 5 | 同上 |
| Chain_OriginFX | 5 | 同上 |
| *_ImpactFX（各类） | 20 | AOE / Pierce 可同时命中多个敌人 |
| *_ImpactFX_Light（被动） | 20 | 被动自动攻击频率较高 |
| ChainArcFX | 10 | 每次 Chain 最多 N 跳，默认 3-4 跳 |
| DeathFX | 30 | 大波次同时死亡（瘟疫/AOE 清场） |
| ChargeFX | 1 | 同时只有一个蓄力实例 |

## 边界情况

| 情况 | 处理方式 |
|------|---------|
| `hits.Length == 0`（未命中） | 不播放任何特效（#8 不广播 OnAttackExecuted，本系统无法收到，不需要额外处理） |
| 游戏处于 Paused / LevelingUp 状态时有事件到达 | 正常播放——游戏暂停时特效已经在飞（技能系统保证不再调用 ExecuteAttack，但正在飞行中的粒子应继续播放直到自然结束）|
| ChainArcPool 耗尽（弧线数 > 10） | 静默跳过多余弧线的显示（不报错），效果降级但不影响逻辑 |
| DeathFX 池耗尽（大量同帧死亡） | 静默跳过，优先给最近死亡的敌人（池满时 Get() 返回 null，调用方检查 null 后跳过） |
| 蓄力中途更换 ClickForm（理论上不可发生） | `_activeChargeFX` 若非 null，在 AddSkill(ClickForm) 时强制归还并置 null，下次点击重新生成 |
| Chain 只有 1 个命中目标（hits.Length == 1） | 播放 Chain_Origin + Chain_Impact，无荷电弧线（条件：`hits.Length >= 2`）|
| `chargeTime == 0` 的即时技能（chargeRatio 始终 = 1.0） | OnChargeStarted 仍触发（技能系统广播），蓄力特效在极短时间内播放后被 OnAttackExecuted 立即回收，视觉上"闪现"一帧，符合即时技能的手感 |
| `OnChargeUpdated(pos, 0)` 收到 ratio = 0（被 LevelingUp 取消） | 立即归还 ChargeFX 池，不等待自然结束 |

## 依赖关系

| 类型 | 系统 | 性质 |
|------|------|------|
| 硬依赖（事件订阅） | 点击攻击系统（#8） | 若无 OnAttackExecuted 事件，本系统无事可做 |
| 硬依赖（事件订阅） | 技能系统（#9） | OnChargeStarted / OnChargeUpdated 蓄力触发 |
| 硬依赖（基础设施） | 对象池系统（#5） | 所有特效实例从池中获取 |
| 软依赖（事件订阅） | 敌人（#10 定义的 EnemyEvents） | 无死亡事件则无死亡特效，其余功能不受影响 |
| 数据读取 | AttackRequest / AttackResult（#8 定义） | `isPassiveAttack`、`attackType`、`hits[]` 字段 |

**下游**：无任何系统依赖本系统（纯消费者，零耦合输出）

## 调节旋钮

| 旋钮 | 默认值 | 安全范围 | 极端行为 |
|------|--------|---------|---------|
| `ARC_DURATION` | 0.25 秒 | 0.1 – 0.5 | <0.1：弧线一闪而逝，看不清；>0.5：弧线滞留太久，视觉拥挤 |
| `MIN_CHARGE_RATE` | 0.3 | 0.1 – 0.8 | <0.1：蓄力初期几乎看不到特效 |
| `MAX_CHARGE_RATE` | 2.5 | 1.5 – 4.0 | >4.0：满蓄力时特效速率过快，闪烁刺眼 |
| `lightScale` | 0.6 | 0.3 – 0.9 | <0.3：被动攻击几乎不可见；>0.9：与主动攻击视觉区分不足 |
| `ChainArcPool容量` | 10 | 5 – 20 | <5：多段 Chain 弧线被截断；>20：内存浪费 |
| `DeathFXPool容量` | 30 | 15 – 60 | <15：大波死亡时部分无特效；>60：内存峰值压力 |

所有旋钮以 `[SerializeField]` 暴露在 Inspector，支持运行时调整。

### 音效系统对接
本系统不产生音效。音效系统（#20）订阅相同的 `OnAttackExecuted` 事件，与本系统并行运行，不存在调用关系。

## UI 需求

无直接 UI 需求。本系统所有内容均为世界空间粒子特效，不依赖 Canvas 或 UI 层。

## 验收标准

1. **Single 命中**：点击命中敌人 → `FX_Origin_Single` 播放于 `result.origin`，`FX_Impact_Single` 播放于 `hit.hitPosition`
2. **AOE 命中**：陨石点击 → Origin 特效播放 0.8 秒，Impact 冲击波在落点扩散，多目标各自有 Impact 特效
3. **Pierce 命中**：圣光洪流 → 垂直光柱沿点击 X 坐标，每个命中敌人有光环闪烁
4. **Chain 命中**：锁链命中 3 个目标 → Origin + 3 个 Impact + 2 条荷电弧线（hits[0]→hits[1]，hits[1]→hits[2]）
5. **Chain 只命中 1 个**：无荷电弧线，只有 Origin + 1 个 Impact
6. **被动攻击**：`isPassiveAttack = true` 时，无 Origin 层，Impact 使用轻量变体（目视明显比主动攻击小）
7. **蓄力特效**：按住鼠标 → 能量聚集效果出现并随按住时长变亮；释放后特效消失，Origin/Impact 接管
8. **蓄力取消**：切到 LevelingUp 状态 → 蓄力特效立即消失（不拖尾）
9. **死亡特效**：击杀敌人 → 死亡位置播放通用死亡爆碎特效
10. **无命中无特效**：点击空白处（#8 不广播事件）→ 无任何特效播放
11. **对象池不泄漏**：运行 5 分钟后，所有对象池大小不超过初始容量上限（无实例累积）
12. **性能**：同帧内 10 个敌人死亡 + 1 次 AOE + 1 次 Chain，帧率不低于 60fps（在目标平台中端 PC 测试）

## 开放问题

| # | 问题 | 优先级 | 备注 |
|---|------|--------|------|
| 1 | 效果类被动（余震、瘟疫）触发的 ExecuteAttack 也会产生特效吗？ | 中 | 当前设计：是的，余震/瘟疫的 `isPassiveAttack = true`，走被动轻量特效路径。这意味着连锁瘟疫传播会有大量轻量死亡特效——视觉上可能有爽感，但也可能刷屏。实现后测试再决定是否加频率限制。 |
| 2 | 荷电弧线是否需要支持弯曲路径（多节点 LineRenderer）还是直线足够？ | 低 | MVP 直线（2 点 LineRenderer）+ 噪声纹理模拟不规则感。若视觉反馈不够好，考虑 3-5 个节点的随机偏移路径。 |
| 3 | 天雷自动攻击是否需要比其他 AutoAttack 更醒目的特效（独立天雷下坠效果）？ | 低 | 当前分类：天雷走 `FX_Impact_Single_Light`，视觉上与玩家主动 Single 相同但更小。若游戏测试中天雷"存在感不足"，可升级为独立下坠轨迹预制件。 |
| 4 | AttackRequest 中 `isPassiveAttack` 字段是否已在 #8 GDD 中补充？ | 高 | 技能系统 GDD 提到此字段，但 #8 GDD 的 `AttackRequest` 结构体未列出。需同步更新 #8 GDD（已知遗留 action item）。 |
