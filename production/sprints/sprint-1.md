# Sprint 1 — 2026-03-29 至 2026-04-04

## 冲刺目标

**实现 MVP 核心框架**：9 个基础系统全部就位，游戏可以从启动运行到胜利/失败判定，形成完整但粗糙的核心循环。

---

## 容量

| 项目 | 数值 |
|------|------|
| 冲刺天数 | 7 天 |
| 人工指导时间 | ~22 小时（工作日 5×2h + 周末 2×6h）|
| 实现方式 | AI 全程执行代码实现，人工负责方向指导、代码审查、验收测试 |

## 任务

### Must Have（关键路径）

| ID | 任务 | 负责 Agent | 预计复杂度 | 依赖 | 验收标准 |
|----|------|-----------|-----------|------|---------|
| S1-01 | 游戏循环管理器：状态机、15 分钟倒计时、胜利/失败判定、事件系统 | gameplay-programmer | 中 | 无 | 5 个状态转换正确；倒计时不显示负数；OnGameStart/Victory/Defeat 事件触发 |
| S1-02 | 输入系统：MouseDown/MouseHeld/MouseUp 事件封装，屏幕坐标→世界坐标 | unity-specialist | 低 | 无 | 点击事件正确广播；蓄力时长可查询；与 UI 输入不冲突 |
| S1-03 | 技能数据库：SkillData ScriptableObject + 5 个基础技能配置（神罚之指 + 4 种形态） | unity-specialist | 低 | 无 | SO 可在 Inspector 中配置；神罚之指等 5 种技能数据正确填写 |
| S1-04 | 敌人数据库：EnemyData ScriptableObject（含 faithValue 字段）+ 5 种敌人配置 | unity-specialist | 低 | 无 | Worker/Elite/Priest/Engineer/Zealot 数据完整；faithValue 字段存在 |
| S1-05 | 对象池系统：通用对象池，支持 Get/Return，预热容量可配置 | gameplay-programmer | 中 | 无 | 100 单位时无 GC 分配；Get/Return 正确复用对象 |
| S1-06 | 塔建造系统：10 层结构、完整度累积、层级解锁、失败事件广播 | gameplay-programmer | 高 | S1-01 | 人类到达后完整度 +N%；第 10 层达 100% 触发 OnTowerCompleted；每层宽度按设计正确缩减 |
| S1-07 | 敌人生成系统：波次生成、从对象池取用、基于游戏进度的难度递增 | gameplay-programmer | 中 | S1-01, S1-04, S1-05 | 游戏开始后敌人正确从屏幕边缘生成；随时间生成频率和单位类型正确变化 |
| S1-08 | 点击攻击系统：伤害计算、碰撞检测（Single/AOE/Chain/Pierce）、AttackResult 广播 | gameplay-programmer | 中 | S1-02 | 4 种 AttackType 均能正确击中目标；伤害数值正确；OnAttackExecuted 事件触发 |
| S1-09 | 技能系统：点击形态管理、蓄力计算、被动倍率、自动攻击计时器（天雷/神火/追踪光球） | gameplay-programmer | 高 | S1-01, S1-02, S1-03, S1-08 | 点击 → 蓄力 → 伤害公式正确；切换 ClickForm 生效；天雷每 3 秒触发；暂停时停止 |
